## 2. 知识点讲解

本节课程演示了如何创建 HCP Terraform 账号。对考试来说，重点不是记住每个网页按钮，而是理解账号、组织和登录入口之间的关系。课程提到 HCP Terraform 托管服务通常通过 `app.terraform.io` 访问，用户需要注册账号、验证邮箱、登录后再创建或加入组织。

账号是用户身份，organization 是团队协作和计费的边界。一个用户可以属于多个 organization。注册账号本身不等于已经有可用于生产的工作空间，后续还需要创建 organization、project、workspace，并根据 workflow 类型连接 VCS 或配置 CLI-driven workflow。

实际练习中可以注册免费账号熟悉界面，但本项目的考试训练不强制你注册，因为真实考试不会要求现场登录 HCP。你需要能判断账号注册只是 HCP Terraform 上手流程的第一步，后续协作、权限、变量和 state 都发生在 organization 和 workspace 等对象中。

## 4. 判断题

1. 注册账号只是使用 HCP Terraform 的第一步，不等于已经配置好 workspace。
2. HCP Terraform 账号和 organization 是同一个概念。
3. 真实考试通常更关注账号、组织、工作区的关系，而不是网页按钮位置。

## 6. 参考答案与解析

选择题：1-A，2-A，3-A。

判断题：1-对，2-错，3-对。

场景题参考：

1. 注册账号只建立用户身份。remote state 是否配置，要看是否创建 workspace、配置 backend 或 `cloud` block，并完成初始化或迁移。
2. 可以学习 organization、project、workspace、CLI-driven workflow、变量配置、run 历史、state 版本和权限概念，不需要真实 AWS 资源。

## 7. 复习提醒

- 账号是身份，organization 是协作和计费边界。
- workspace 才是很多 Terraform 运行、变量和 state 行为的核心对象。
- 不要把“能登录平台”和“已完成基础设施工作流”混为一谈。
