# Terraform 实操训练 11：File System Mirror 概念

## 1. 课程元数据

- 课程编号：11
- 来源文件：`doc/11.txt`

- 是否可用 Docker 运行：`yes`
- 核心主题：file system mirror、离线 provider 安装、`TF_CLI_CONFIG_FILE`
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

File system mirror 解决的是离线或受限网络环境下 provider 安装问题。Terraform 默认访问 `registry.terraform.io`，但在内网、涉密或受控网络中，runner 可能无法访问公网。

通过 `terraform providers mirror` 可以把 provider 包下载到本地 mirror，再通过 CLI 配置的 `provider_installation` 指向该 mirror。这样 `terraform init` 可以从本地目录安装 provider。

## 3. 实验设计

### 背景

企业 CI runner 不能访问公网 registry。

### 问题

没有本地 mirror 时，`terraform init` 会失败。

### 任务

生成 provider mirror，并通过 `TF_CLI_CONFIG_FILE` 使用 file system mirror 初始化。

### 预期结果

`terraform init` 从 mirror 安装 `hashicorp/local`，并成功完成本地文件实验。

## 5. 运行手册

```sh
cd /workspace/practice/labs/11
terraform providers mirror /workspace/practice/labs/11/mirror
rm -rf .terraform .terraform.lock.hcl
TF_CLI_CONFIG_FILE=/workspace/practice/labs/11/terraform-cli.rc terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

关键结果：

- `terraform providers mirror` 成功下载 `hashicorp/local v2.9.0`。
- `TF_CLI_CONFIG_FILE` 指向 `terraform-cli.rc` 后，`terraform init` 成功。
- 初始化时显示 provider 从 mirror 安装，签名状态为 `unauthenticated`，这是 file system mirror 的常见提示。
- `terraform test` 返回 `1 passed, 0 failed`。

## 7. 排障指南

1. mirror 目录为空：先执行 `terraform providers mirror`。
2. `terraform.rc` 路径无效：确认容器内路径是 `/workspace/practice/labs/11/...`。
3. 多平台校验和警告：按提示用 `terraform providers lock -platform=...` 补齐。

terraform destroy -auto-approve
