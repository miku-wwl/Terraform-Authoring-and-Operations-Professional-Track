# Terraform 考试训练 131：迁移 State 到 HCP Terraform

## 1. 课程元数据

- 课程编号：131
- 来源文件：`doc/131.txt`
- 环境分类：`theory`
- 是否可用 Docker 运行：`no`
- 核心主题：HCP Terraform remote state、state version 和本地 state 迁移
- 前置条件：理解 Terraform state、backend、`terraform login`、`terraform init`
- 训练方式：知识点讲解 + 选择题 + 判断题 + 场景题

## 2. 知识点讲解

HCP Terraform 可以作为安全的远程 state 存储位置，并提供 state 版本历史、协作访问和平台化管理。课程中演示了把本地 state 迁移到 HCP Terraform workspace 的流程。核心步骤是：先在配置中添加 `cloud` block，指定 organization 和 workspace；执行 `terraform login` 完成认证；再运行 `terraform init`，按提示迁移已有 state。

State 迁移是高风险操作，生产环境必须先备份本地 `terraform.tfstate`。迁移后，本地 state 文件可能不再保存完整资源 JSON，真实 state 由 HCP Terraform 管理。HCP Terraform 可以保留 state version，方便回看历史变化，但这不等于可以随意忽略备份和变更流程。

考试还会考两个命令细节。自动化迁移时，可以使用 `terraform init -migrate-state` 避免交互式确认。若本地 Terraform 版本与远端 workspace 版本不一致，可能遇到版本检查问题；在确认兼容的前提下，课程提到可以使用 `-ignore-remote-version`，但生产中应优先保持版本一致。

## 3. 考试选择题

1. 迁移本地 state 到 HCP Terraform 前，最重要的安全动作是什么？
   - A. 备份现有 `terraform.tfstate`
   - B. 删除所有 `.tf` 文件
   - C. 清空 Git 历史
   - D. 关闭所有 workspace

2. `terraform login` 在 state 迁移流程中的作用是什么？
   - A. 认证到 HCP Terraform
   - B. 格式化 Terraform 代码
   - C. 创建本地 Docker 网络
   - D. 自动编写 Sentinel policy

3. 自动化环境中迁移 state 时，哪个参数可以减少交互式确认？
   - A. `-migrate-state`
   - B. `-no-color`
   - C. `-json`
   - D. `-check`

## 4. 判断题

1. 迁移 state 前应备份本地 state 文件。
2. HCP Terraform 的 remote state 有助于团队协作和 state 历史管理。
3. 遇到远端版本不一致时，应该无条件使用 `-ignore-remote-version`。

## 5. 场景题

1. 团队最初使用本地 state，现在要迁移到 HCP Terraform 以便多人协作。请描述关键步骤。

2. 初始化迁移时报错：本地 Terraform 版本不满足远端 workspace 要求。你会如何处理？

## 6. 参考答案与解析

选择题：1-A，2-A，3-A。

判断题：1-对，2-对，3-错。

场景题参考：

1. 先备份 `terraform.tfstate`，在配置中添加 `cloud` block 指定 organization 和 workspace，执行 `terraform login`，再运行 `terraform init` 并确认迁移。自动化场景可使用 `terraform init -migrate-state`。
2. 优先统一本地和远端 Terraform 版本。如果确认配置兼容且确有必要，才考虑 `-ignore-remote-version`，并记录风险。

## 7. 复习提醒

- State 迁移先备份。
- `cloud` block 指目标，`terraform login` 解决身份，`terraform init` 执行初始化和迁移。
- `-migrate-state` 适合自动化；`-ignore-remote-version` 不是无脑默认项。
