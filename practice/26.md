# Terraform 实操训练 26：Heredoc 多行字符串

## 1. 课程元数据

- 课程编号：26
- 来源文件：`doc/26.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：heredoc、多行字符串、脚本生成
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Heredoc 是 Terraform 中维护多行字符串的主要方式。相比大量 `\n` 拼接，heredoc 更适合脚本、配置文件、JSON/YAML 片段和说明文本。

生产中要关注缩进是否会被保留、变量插值是否符合预期，以及生成文件是否被下游工具正确解析。

## 3. 实验设计

背景：平台团队需要用 Terraform 生成部署脚本。

问题：用普通字符串维护脚本可读性差。

任务：使用 heredoc 生成 `deploy.sh`。

预期结果：脚本包含 shebang 和多行命令。

## 5. 运行手册

```sh
cd /workspace/practice/labs/26
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

已执行 `init/fmt/validate/test/plan/apply/output/destroy`。`terraform test` 返回 `1 passed, 0 failed`，生成的脚本包含 `#!/bin/sh` 和多行部署内容。

## 7. 排障指南

1. 结束标记报错：开始和结束 delimiter 必须一致。
2. 缩进不符合预期：基础 heredoc 会保留前导空白。
3. 插值被执行：需要字面量 `${}` 时要额外转义。
4. 脚本不可执行：检查 `file_permission`。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf output .terraform .terraform.lock.hcl
```
