- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

`terraform init` 会下载 provider plugin。多个项目重复下载相同 provider 会浪费时间和带宽。Plugin cache 可以把 provider 缓存在共享目录中，后续项目初始化时复用相同版本。

缓存不是替代 lock file。生产环境仍应关注 `.terraform.lock.hcl`，因为它负责记录 provider 版本和校验和。

## 3. 实验设计

### 背景

平台团队有大量 Terraform 项目，初始化速度慢。

### 问题

每个项目重复从 registry 下载 provider，浪费 CI 时间。

### 任务

配置 `TF_PLUGIN_CACHE_DIR`，生成 plugin cache 策略文档，并验证命令记录。

### 预期结果

生成 `output/plugin-cache-policy.md`。

## 5. 运行手册

```sh
cd /workspace/practice/labs/9
mkdir -p .terraform-plugin-cache
export TF_PLUGIN_CACHE_DIR=/workspace/practice/labs/9/.terraform-plugin-cache
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

关键结果：

- 共享缓存目录 `.terraform-plugin-cache` 创建成功。
- `terraform test` 返回 `1 passed, 0 failed`。
- `plan/apply/destroy` 全链路通过。

## 7. 排障指南

1. 缓存目录不存在：先执行 `mkdir -p .terraform-plugin-cache`。
2. 仍然重复下载：检查 `TF_PLUGIN_CACHE_DIR` 是否在当前 shell 生效。
3. 校验和问题：检查 `.terraform.lock.hcl`。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -rf .terraform .terraform-plugin-cache output tfplan terraform.tfstate*
```
