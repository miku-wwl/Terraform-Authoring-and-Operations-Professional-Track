# Terraform 实操训练 17：Code Scan 与 Plan Scan

## 1. 课程元数据

- 课程编号：17
- 来源文件：`doc/17.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：源码扫描、plan 扫描、Checkov、变量注入风险
- 前置条件：Docker、`hashicorp/terraform:1.11`、`bridgecrew/checkov:latest`

## 2. 知识点讲解

源码扫描能在提交早期发现问题，但它看到的是 HCL 源码，不一定知道最终变量值。plan 扫描看到的是 Terraform 计算后的实际变更，因此能发现运行时注入的危险值。

本实验用 `host_network_enabled` 变量演示：源码中没有写死 `host_network = true`，源码扫描可能看不到最终风险；但 plan 阶段通过 `-var` 注入后，plan 扫描能发现 Pod 使用宿主机网络。

## 3. 实验设计

### 背景

团队已经有源码扫描，但仍担心变量或流水线参数绕过检查。

### 问题

危险的宿主机网络配置不在源码中固定写死，而是在 plan 阶段注入。

### 任务

生成 plan JSON，并分别用 Checkov 扫源码和 plan。

### 预期结果

源码扫描中宿主机网络检查通过，plan 扫描中对应 Kubernetes 检查失败。

## 5. 运行手册

```sh
cd /workspace/practice/labs/17
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan -var 'host_network_enabled=true'
terraform show -json tfplan > plan.json
```

源码扫描：

```powershell
docker run --rm `
  -v "${PWD}/practice/labs/17:/tf" `
  bridgecrew/checkov:latest `
  -d /tf --framework terraform --check CKV_K8S_19 --soft-fail --compact
```

plan 扫描：

```powershell
docker run --rm `
  -v "${PWD}/practice/labs/17:/tf" `
  bridgecrew/checkov:latest `
  -f /tf/plan.json --framework terraform_plan --check CKV_K8S_19 --soft-fail --compact
```

## 6. 验证报告

状态：通过。

关键结果：

- Terraform `init/fmt/validate/test/plan` 成功。
- 未执行 `apply`，不需要 Kubernetes 集群。
- 源码扫描限定 `CKV_K8S_19` 时没有失败结果。
- plan 扫描限定 `CKV_K8S_19` 时返回 `Failed checks: 1`。
- plan 扫描能看到变量注入后的 `host_network = true`。
- 这证明 plan scan 能发现变量注入后的真实风险。

## 7. 排障指南

1. `plan` 要求变量：补 `-var 'host_network_enabled=true'`。
2. Checkov 找不到 `plan.json`：先执行 `terraform show -json tfplan > plan.json`。
3. 不要执行 apply：这是扫描实验，不需要真实集群。

## 8. 清理方式

```sh
rm -rf .terraform tfplan plan.json terraform.tfstate*
```
