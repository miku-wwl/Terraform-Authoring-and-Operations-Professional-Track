## 2. 知识点讲解

本节强调 Terraform 专业能力不只是会写 `.tf`，还要熟悉 CLI 命令和参数。不同 Terraform 版本支持的参数可能变化，因此在实操和考试中，`terraform <command> -help` 是最快速、最贴近当前版本的查询方式。

生产环境里，CLI 参数决定了流水线是否可审计、可复现、可自动失败。例如 `-input=false` 控制是否等待人工输入，`-no-color` 控制日志是否适合机器解析，`-out=tfplan` 控制 plan 与 apply 是否一致。

## 3. 实验设计

### 背景

平台团队要为 CI/CD runner 建立 Terraform 命令速查表，避免工程师临时查浏览器文档时忽略当前 CLI 版本。

### 问题

团队只记得 `init/plan/apply`，但不熟悉各命令参数，导致自动化脚本经常遗漏必要 flag。

### 任务

生成一份本地 CLI runbook，并用测试验证关键 help 命令和自动化参数被记录。

### 预期结果

生成 `output/terraform-cli-runbook.md`，并通过 `terraform test`。

## 5. 运行手册

```sh
cd /workspace/practice/labs/6
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

已执行：`init/fmt/validate/test/plan/apply/output/destroy`。

关键结果：

- `terraform test` 返回 `1 passed, 0 failed`。
- `terraform plan` 显示 `Plan: 1 to add, 0 to change, 0 to destroy`。
- `terraform apply` 生成 CLI 速查文档。
- `terraform destroy` 成功销毁本地文件资源。

## 7. 排障指南

1. 忘记参数含义：先执行 `terraform <command> -help`。
2. `terraform test` 失败：检查 `outputs.tf` 是否仍暴露 `command_catalog` 和 `automation_flags`。
3. 计划输出不适合归档：确认命令包含 `-no-color`。

## 8. 清理方式

```sh
terraform destroy -auto-approve
```
