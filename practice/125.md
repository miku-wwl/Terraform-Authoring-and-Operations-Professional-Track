# Terraform 考试训练 125：Variable Sets

## 1. 课程元数据

- 课程编号：125
- 来源文件：`doc/125.txt`
- 环境分类：`theory`
- 是否可用 Docker 运行：`no`
- 核心主题：HCP Terraform variable sets 的复用、作用域和敏感变量管理
- 前置条件：理解 Terraform variable、environment variable、workspace variable
- 训练方式：知识点讲解 + 选择题 + 判断题 + 场景题

## 2. 知识点讲解

Variable set 用来解决多个 workspace 重复维护相同变量的问题。没有 variable set 时，如果三个 workspace 都需要相同的区域、账号标识、审批开关或凭据变量，就必须在每个 workspace 单独配置。一旦变量变更，维护成本和遗漏风险都会增加。

在 HCP Terraform 中，variable set 可以集中定义一组变量，并把它关联到 organization、project 或特定 workspace 范围。这样多个 workspace 可以共享同一批变量。变量可以是 Terraform variable，也可以是 environment variable；敏感变量应标记为 sensitive，避免在界面和日志中暴露。

考试常见陷阱是把 variable set 和单个 workspace variable 混为一谈。Workspace variable 只服务单个 workspace；variable set 是复用机制，适合跨 workspace 的公共配置。另一个陷阱是忘记变量作用域，导致以为所有 variable set 一定全局生效。

## 3. 考试选择题

1. Variable set 的主要价值是什么？
   - A. 在多个 workspace 间复用一组变量
   - B. 替代所有 Terraform resource
   - C. 自动创建 Git 仓库
   - D. 禁止使用 sensitive variable

2. 哪类变量更适合标记为 sensitive？
   - A. 数据库密码或云访问密钥
   - B. 普通环境名称 `dev`
   - C. README 标题
   - D. Terraform 文件名

3. 如果多个 workspace 需要相同的环境变量，最合适的 HCP Terraform 功能是什么？
   - A. variable set
   - B. `terraform fmt`
   - C. `terraform graph`
   - D. provider mirror

## 4. 判断题

1. Variable set 可以减少重复配置变量的维护成本。
2. Sensitive variable 的目的之一是降低敏感值在 UI 或日志中暴露的风险。
3. Variable set 一旦创建，就只能绑定一个 workspace，不能用于复用。

## 5. 场景题

1. 三个应用 workspace 都使用同一组云区域、账号 ID 和通知邮箱。后续邮箱变更时，团队不希望逐个 workspace 修改。你会推荐什么能力？

2. 开发者把 access key 作为普通变量明文写在仓库中。请指出风险和更合适的处理方式。

## 6. 参考答案与解析

选择题：1-A，2-A，3-A。

判断题：1-对，2-对，3-错。

场景题参考：

1. 推荐 variable set，把共享变量集中维护，并按 organization、project 或 workspace 范围关联。
2. 风险是密钥泄露和审计困难。应改用 HCP Terraform 变量或 variable set，并标记为 sensitive；真实密钥不应提交到 Git。

## 7. 复习提醒

- Workspace variable 是单 workspace 配置，variable set 是复用配置。
- 区分 Terraform variable 和 environment variable。
- 看到“多个 workspace 重复变量”，优先想到 variable set。
