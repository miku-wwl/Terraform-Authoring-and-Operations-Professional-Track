# Terraform 实操训练 36：输入变量验证基础

## 1. 课程元数据

- 课程编号：36
- 来源文件：`doc/36.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：输入变量验证、plan 阶段失败、命名约束
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

输入变量验证把错误尽早暴露在 Terraform 计划阶段，避免远端 API 到 apply 时才拒绝请求。生产环境中，命名规则、规格白名单、环境枚举和容量范围都应尽量前置验证。

## 3. 实验设计

背景：平台团队要求工作区名称符合统一命名规范。

问题：错误命名如果到 apply 才失败，会浪费流水线时间。

任务：用 `validation` 和 `regex` 限制名称。

预期结果：合法名称通过，非法名称在 plan 阶段失败。

## 4. 实验文件

- `practice/labs/36/main.tf`
- `practice/labs/36/tests/input_validation_basics.tftest.hcl`
- `practice/labs/36/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/36
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。已执行完整 Terraform 流程，`terraform test` 返回 `1 passed, 0 failed`。

## 7. 排障指南

1. 正则不匹配：先用简单样例验证表达式。
2. 错误信息不清楚：`error_message` 要告诉用户如何修复。
3. 中文或大写被拒绝：这是本实验命名规则的预期结果。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
