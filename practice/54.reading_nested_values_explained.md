# Terraform 实操训练 54 讲解：读取嵌套值

## 1. 这一节到底在学什么？

Lab 54 学的是 Terraform 里如何**读取嵌套类型中的具体值**。

上一节 Lab 53 讲的是嵌套类型怎么建模，比如：

```hcl
map(object({
  ports = list(object({
    name = string
    port = number
  }))
  tags = map(string)
}))
```

这一节更进一步：  
不是只会声明这种复杂类型，而是要能准确取到里面的值。

一句话理解：

> 读取嵌套值，就是沿着数据结构一层一层往下访问，直到拿到你真正需要的字段。

---

## 2. 为什么读取嵌套值很重要？

Terraform 真实项目里经常不是简单变量：

```hcl
var.name
```

而是复杂结构：

```hcl
var.services["api"].ports[0].port
```

如果你不会读嵌套值，就会在这些地方卡住：

- 模块输入是 `map(object)`
- 服务配置里有多个端口
- 环境配置里有多层字段
- JSON decode 后得到复杂对象
- data source 返回复杂结构
- output 要从嵌套结构里提取字段
- `for_each` 里要访问 `each.value.xxx`

所以这节是 Terraform 类型系统训练里非常实用的一节。

---

## 3. 读取嵌套值的核心规则

Terraform 读取嵌套值主要靠三种访问方式：

| 数据结构 | 访问方式 | 示例 |
|---|---|---|
| object | `.field` | `var.app.name` |
| map | `["key"]` | `var.services["api"]` |
| list | `[index]` | `var.ports[0]` |

复杂结构就是这三种方式组合起来。

例如：

```hcl
var.services["api"].ports[0].port
```

可以拆成：

1. `var.services`：一个 map。
2. `["api"]`：取 key 为 api 的服务对象。
3. `.ports`：取服务对象里的 ports 字段。
4. `[0]`：取第一个端口对象。
5. `.port`：取端口对象里的 port 字段。

这就是“顺着结构一层层读”。

---

## 4. object 字段读取

最基础的是 object 字段读取。

示例：

```hcl
locals {
  app = {
    name     = "payments"
    port     = 8080
    replicas = 2
  }
}
```

读取字段：

```hcl
local.app.name
local.app.port
local.app.replicas
```

结果分别是：

```text
payments
8080
2
```

固定字段通常用点号访问最清晰。

---

## 5. map key 读取

map 通过 key 读取。

示例：

```hcl
locals {
  env_sizes = {
    dev  = "small"
    test = "medium"
    prod = "large"
  }
}
```

读取 dev：

```hcl
local.env_sizes["dev"]
```

结果：

```text
small
```

如果 key 来自变量：

```hcl
local.env_sizes[var.environment]
```

这就是动态 key 访问。

注意：如果 key 不存在，会报错。  
如果希望带默认值，可以用：

```hcl
lookup(local.env_sizes, var.environment, "small")
```

---

## 6. list 索引读取

list 通过索引读取，索引从 0 开始。

示例：

```hcl
locals {
  ports = [80, 443, 8080]
}
```

读取第一个端口：

```hcl
local.ports[0]
```

结果：

```text
80
```

读取第二个端口：

```hcl
local.ports[1]
```

结果：

```text
443
```

注意：  
如果 list 只有 3 个元素，合法索引是 `0`、`1`、`2`。  
访问 `[3]` 会越界。

---

## 7. object 里套 object

示例：

```hcl
locals {
  app = {
    name = "payments"

    resources = {
      cpu    = "500m"
      memory = "512Mi"
    }
  }
}
```

读取 CPU：

```hcl
local.app.resources.cpu
```

读取 memory：

```hcl
local.app.resources.memory
```

这就是连续点号访问。

---

## 8. object 里套 list

示例：

```hcl
locals {
  app = {
    name  = "payments"
    ports = [80, 443, 8080]
  }
}
```

读取第一个端口：

```hcl
local.app.ports[0]
```

读取第三个端口：

```hcl
local.app.ports[2]
```

路径拆解：

```text
local.app       -> object
.ports          -> list
[0]             -> 第一个元素
```

---

## 9. object 里套 map

示例：

```hcl
locals {
  app = {
    name = "payments"

    tags = {
      Owner       = "platform"
      Environment = "dev"
    }
  }
}
```

读取 Owner：

```hcl
local.app.tags["Owner"]
```

读取 Environment：

```hcl
local.app.tags["Environment"]
```

路径拆解：

```text
local.app       -> object
.tags           -> map
["Owner"]       -> map key
```

---

## 10. list(object) 里读取字段

示例：

```hcl
locals {
  ports = [
    {
      name = "http"
      port = 8080
    },
    {
      name = "metrics"
      port = 9090
    }
  ]
}
```

读取第一个端口名称：

```hcl
local.ports[0].name
```

读取第一个端口号：

```hcl
local.ports[0].port
```

读取第二个端口号：

```hcl
local.ports[1].port
```

路径拆解：

```text
local.ports     -> list
[1]             -> 第二个 object
.port           -> object 字段
```

---

## 11. map(object) 里读取字段

示例：

```hcl
locals {
  services = {
    api = {
      port     = 8080
      replicas = 2
    }

    worker = {
      port     = 9090
      replicas = 1
    }
  }
}
```

读取 api 端口：

```hcl
local.services["api"].port
```

读取 worker 副本数：

```hcl
local.services["worker"].replicas
```

路径拆解：

```text
local.services      -> map
["api"]             -> key 为 api 的 object
.port               -> object 字段
```

这种写法在生产 Terraform 里非常常见。

---

## 12. 更复杂的嵌套读取

假设有：

```hcl
locals {
  services = {
    api = {
      image = "api:v1"

      ports = [
        {
          name = "http"
          port = 8080
        },
        {
          name = "metrics"
          port = 9090
        }
      ]

      tags = {
        Environment = "dev"
        Owner       = "platform"
      }
    }
  }
}
```

读取 api 镜像：

```hcl
local.services["api"].image
```

读取 api 第一个端口号：

```hcl
local.services["api"].ports[0].port
```

读取 api 第二个端口名称：

```hcl
local.services["api"].ports[1].name
```

读取 api 的 Owner tag：

```hcl
local.services["api"].tags["Owner"]
```

只要你能拆清楚每一层是什么类型，就不会乱。

## 16. 如何快速判断访问路径？

面对复杂结构，按这个方法拆：

### 第一步：确认根对象

例如：

```hcl
var.services
```

它是 list、map 还是 object？

### 第二步：判断下一步访问方式

如果是 map：

```hcl
["key"]
```

如果是 list：

```hcl
[index]
```

如果是 object：

```hcl
.field
```

### 第三步：继续往下拆

例如：

```hcl
var.services["api"].ports[0].port
```

拆成：

```text
var.services         map
["api"]              object
.ports               list
[0]                  object
.port                number
```

这样就不会乱。

## 17. `for_each` 里的嵌套读取

如果有：

```hcl
variable "services" {
  type = map(object({
    image = string
    port  = number
  }))
}
```

资源里：

```hcl
resource "local_file" "service" {
  for_each = var.services

  filename = "${path.module}/output/${each.key}.txt"
  content  = "image=${each.value.image}, port=${each.value.port}"
}
```

这里：

```hcl
each.key
```

是 map 的 key，例如 `api`。

```hcl
each.value
```

是 key 对应的 object。

```hcl
each.value.image
```

是 object 里的 image 字段。

```hcl
each.value.port
```

是 object 里的 port 字段。

这就是生产 Terraform 里最常见的嵌套读取场景之一。

## 18. `for` 表达式里的嵌套读取

如果有：

```hcl
locals {
  ports = [
    {
      name = "http"
      port = 8080
    },
    {
      name = "metrics"
      port = 9090
    }
  ]
}
```

提取所有端口号：

```hcl
locals {
  port_numbers = [
    for p in local.ports : p.port
  ]
}
```

结果：

```hcl
[8080, 9090]
```

提取所有端口名：

```hcl
locals {
  port_names = [
    for p in local.ports : p.name
  ]
}
```

结果：

```hcl
["http", "metrics"]
```

这里的 `p` 就是 list 中的每一个 object。

## 20. 常见坑

### 坑 1：list 索引越界

如果 list 只有一个元素：

```hcl
var.ports[0]
```

可以。

但：

```hcl
var.ports[1]
```

会失败。

解决思路：

- 确认 `length(var.ports)`
- 用 validation 保证长度
- 用 `for` 表达式处理全部元素，不硬编码索引

---

### 坑 2：map key 不存在

如果 map 里没有：

```hcl
"api"
```

你访问：

```hcl
var.services["api"]
```

会失败。

可以用：

```hcl
lookup(var.services, "api", null)
```

但生产里更好的做法是明确校验 key 是否存在。

---

### 坑 3：object 字段写错

如果字段叫：

```hcl
replicas
```

你写：

```hcl
replica
```

会失败。

这类错误通常在 `terraform validate` 或 `terraform test` 阶段暴露。

---

### 坑 4：把 map 当 object

如果 key 是动态的，应该用：

```hcl
var.services[var.service_name]
```

不要写死成：

```hcl
var.services.service_name
```

后者表示找一个名为 `service_name` 的字段，不是使用变量值。

---

### 坑 5：嵌套太深导致可读性差

如果访问路径长成这样：

```hcl
var.platform.environments[var.env].services[var.service].ports[0].rules[0].target
```

可读性会下降。

可以用 locals 拆开：

```hcl
locals {
  current_env     = var.platform.environments[var.env]
  current_service = local.current_env.services[var.service]
  first_port      = local.current_service.ports[0]
}
```

然后：

```hcl
local.first_port.rules[0].target
```

这样更容易读，也更容易排障。

---

## 21. 用 locals 简化嵌套读取

复杂路径建议用 `locals` 拆解。

原始写法：

```hcl
var.services["api"].ports[0].port
```

可以拆成：

```hcl
locals {
  api_service = var.services["api"]
  first_port  = local.api_service.ports[0]
  api_port    = local.first_port.port
}
```

这有几个好处：

1. 路径更短。
2. 输出更容易读。
3. 测试更容易写。
4. 排障更容易定位是哪一层错了。
5. 后续表达式更清楚。

Terraform 不是代码越短越好。  
工程里可读性很重要。

---

## 22. 变量校验可以保护嵌套读取

如果你必须读取：

```hcl
var.ports[0]
```

最好确保 list 非空：

```hcl
validation {
  condition     = length(var.ports) > 0
  error_message = "ports must contain at least one item."
}
```

如果你必须读取：

```hcl
var.services["api"]
```

最好确保 key 存在：

```hcl
validation {
  condition     = contains(keys(var.services), "api")
  error_message = "services must contain an api entry."
}
```

这样错误会更早、更清楚。

## 23. 输出嵌套值

可以输出某个嵌套字段：

```hcl
output "api_port" {
  value = var.services["api"].ports[0].port
}
```

也可以输出摘要：

```hcl
output "api_summary" {
  value = "${var.services["api"].image}:${var.services["api"].ports[0].port}"
}
```

生产里建议输出必要信息，不要无脑输出整个复杂对象，尤其里面可能有 sensitive 字段。

## 24. Terraform test 在这里有什么价值？

Lab 54 的验证报告里说：

```text
terraform test 返回 1 passed, 0 failed
```

对读取嵌套值来说，测试特别有价值。

因为复杂结构最容易出现：

- 字段路径写错
- 索引错
- key 错
- 输出名称错
- 类型不匹配
- plan 阶段和 apply 阶段值不同

测试可以把这些问题固定下来。

例如测试可能会断言：

- `api_port == 8080`
- `first_service_name == "api"`
- `owner_tag == "platform"`
- `port_count == 2`

这比只看输出靠谱。

## 26. SRE / Platform Engineering 视角

从平台工程角度看，读取嵌套值是日常高频动作。

例如你设计一个服务 catalog：

```hcl
services = {
  api = {
    image = "api:v1"
    ports = [
      {
        name = "http"
        port = 8080
      }
    ]
    tags = {
      Owner = "platform"
    }
  }
}
```

模块里必须能准确读取：

```hcl
each.key
each.value.image
each.value.ports[0].port
each.value.tags["Owner"]
```

这就是平台模块把“配置数据”变成“实际资源”的过程。

不会读嵌套值，就很难写真正可复用的 Terraform 模块。

## 27. 面试/考试记忆版

可以这样记：

- object 字段用 `.field`
- map key 用 `["key"]`
- list 元素用 `[index]`
- 复杂路径就是三者组合
- Terraform list 索引从 0 开始
- key 不存在会报错
- 索引越界会报错
- 复杂路径可以用 locals 拆开
- 嵌套读取最好配合 validation 和 terraform test
- `for_each` 中常用 `each.key` 和 `each.value.field`

一句话总结：

> 读取嵌套值的关键，是先判断每一层的数据类型，再选择 `.field`、`["key"]` 或 `[index]`。
