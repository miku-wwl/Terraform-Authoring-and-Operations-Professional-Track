# Terraform 考试训练 118：HCP Terraform 定价与功能层级讲解版

## 1. 本节要解决什么问题？

这一节不是让你背 HCP Terraform 具体多少钱，也不是让你记住某个套餐的完整功能表。

真正要掌握的是：

> HCP Terraform 是一个平台，不是一个“无限免费”的 Terraform CLI 外壳。考试会考你能不能判断哪些能力属于基础使用，哪些能力属于企业治理、合规和平台化能力。

所以本节的核心判断标准是：

- 本地 Terraform CLI 能做什么？
- HCP Terraform 作为远程平台额外提供什么？
- 哪些能力通常和更高层级计划、企业治理或 Terraform Enterprise 有关？
- 遇到 pricing / plan / governance / compliance 相关题目时，应该怎么排除干扰项？

---

## 2. HCP Terraform 的定位

HCP Terraform 可以理解为 Terraform 的托管平台版本。

Terraform CLI 本身主要负责：

- 读取 `.tf` 配置
- 初始化 provider
- 生成 plan
- 执行 apply
- 管理 state
- 做基础校验和格式化

而 HCP Terraform 更偏平台化，常见能力包括：

- 远程 workspace
- remote state 管理
- remote run
- 团队协作
- 权限控制
- 变量集中管理
- VCS 集成
- run history
- policy governance
- drift detection
- continuous validation
- audit logging
- 企业级合规能力

一句话：

> CLI 是执行工具，HCP Terraform 是协作和治理平台。

考试题里只要看到“团队、审批、审计、策略、漂移检测、持续验证、企业隔离环境”，基本就不是在问普通 CLI 功能了。

---

## 3. 定价与功能层级的正确理解

HCP Terraform 的具体价格、计划名称、功能边界都可能变化。考试通常不会要求你死背某个价格数字。

更稳定的理解是：

| 类型 | 更接近的能力 | 考试判断 |
|---|---|---|
| 基础 CLI 能力 | `fmt`、`validate`、`plan`、`apply`、`console` | 本地 Terraform 就能做 |
| 基础平台能力 | workspace、remote state、remote run、VCS 集成 | HCP Terraform 托管平台能力 |
| 企业治理能力 | audit logging、policy governance、team permissions、drift detection、continuous validation | 通常和高级计划或企业场景有关 |
| 企业自托管能力 | Terraform Enterprise、airgap installation、私有网络隔离部署 | 强合规、强隔离企业场景 |

最重要的是：

> 不要把“我能打开 HCP Terraform UI”理解成“所有平台能力都免费、无限、默认可用”。

个人学习账号可以用来理解概念，但企业真实落地时，必须考虑：

- 组织规模
- 托管资源数量
- workspace 数量
- 用户和团队权限
- 审计要求
- 合规要求
- 是否需要策略控制
- 是否需要漂移检测
- 是否需要自托管或 airgap 环境

---

## 4. HCP Terraform 和 Terraform Enterprise 的区别

这一点很容易考。

### HCP Terraform

HCP Terraform 是 HashiCorp 提供的托管服务。平台运行、升级、服务维护主要由 HashiCorp 负责。

适合场景：

- 想快速使用 Terraform 平台能力
- 不想自己维护平台基础设施
- 可以接受托管服务模式
- 需要团队协作和远程 run

### Terraform Enterprise

Terraform Enterprise 更偏企业自托管版本。企业自己负责部署、维护、升级和运行环境。

适合场景：

- 强隔离环境
- 高合规要求
- 内网部署
- airgap installation
- 企业不允许核心平台依赖公网托管服务

考试判断：

> 看到 airgap、offline、isolated environment、self-hosted、private datacenter，优先想到 Terraform Enterprise，而不是普通 HCP Terraform 免费账号。

## 5. 高频考试关键词

### 5.1 audit logging

audit logging 是审计日志。

它回答的问题是：

- 谁改了 workspace？
- 谁触发了 run？
- 谁改了变量？
- 谁批准了 apply？
- 什么时间发生了什么平台操作？

这不是普通 `terraform plan` 的能力，而是平台治理能力。

### 5.2 drift detection

drift detection 是漂移检测。

漂移的意思是：

> Terraform state 里记录的资源状态，和云上真实资源状态不一致。

比如：

- Terraform 创建了一个安全组
- 某个人手动去云控制台改了安全组规则
- Terraform 配置和真实资源开始不一致
- 这就是 drift

普通 CLI 可以通过 `terraform plan` 发现差异，但 HCP Terraform 的 drift detection 更偏平台自动化检测能力。

考试里不要把二者混成一个东西：

- `terraform plan`：手动或流水线触发，发现配置、state、真实资源之间的差异
- drift detection：平台定期或自动检测漂移，并把它作为治理事件暴露出来

### 5.3 continuous validation

continuous validation 是持续验证。

它的目标不是只在 apply 的时候检查一次，而是持续确认基础设施是否仍然符合预期。

可以理解为：

> 不只是“部署前检查”，而是“部署后也继续盯着”。

它更接近平台治理能力，而不是普通本地 CLI 命令。

### 5.4 policy governance

policy governance 是策略治理。

它解决的是：

- 不允许创建公网暴露的数据库
- 不允许使用不合规的 region
- 不允许创建未打 tag 的资源
- 不允许超出预算的资源规格
- 不允许违反安全基线的配置进入 apply

这类能力通常和 Sentinel、OPA 或平台策略控制有关。

考试判断：

> 只要题目强调“组织统一规则”“强制策略”“合规检查”“审批前阻断”，就属于平台治理，不是普通 `terraform fmt` 或 `validate`。

### 5.5 airgap installation

airgap installation 指强隔离、无公网依赖或极少公网依赖的部署方式。

典型场景：

- 金融机构
- 政府
- 军工
- 强监管企业
- 内网数据中心
- 不允许连接公网 SaaS 的环境

这更接近 Terraform Enterprise 自托管场景。

不要选成：

- 免费 workspace 默认功能
- Docker 本地练习
- `terraform validate` 的功能
- 普通 HCP Terraform UI 设置项

## 6. 考试最容易挖坑的地方

### 坑 1：以为 HCP Terraform 所有能力永久免费

错误理解：

> 我能注册账号，所以所有功能都能免费用。

正确理解：

> HCP Terraform 有不同计划和功能层级。免费或基础能力不代表企业治理能力也无限可用。

### 坑 2：把 Terraform CLI 和 HCP Terraform 平台混为一谈

错误理解：

> `terraform plan` 能发现 drift，所以 drift detection 就是普通 CLI 免费功能。

正确理解：

> CLI 可以通过 plan 发现差异，但平台级 drift detection 是自动化、集中化、治理化能力。

### 坑 3：把 HCP Terraform 和 Terraform Enterprise 混为一谈

错误理解：

> HCP Terraform 和 Terraform Enterprise 只是名字不同，部署方式一样。

正确理解：

> HCP Terraform 是托管服务；Terraform Enterprise 更偏企业自托管。运营责任和适用场景不同。

### 坑 4：背价格，而不是背能力分类

错误理解：

> 考试会问某功能每月多少钱。

正确理解：

> 考试更常考功能归类和场景判断。价格和套餐名称可能变化，不适合死背。

## 7. 选择题讲解

### 题目 1

关于 HCP Terraform 定价，哪项理解最稳妥？

- A. 所有功能永久免费
- B. 所有功能只能购买企业版才能使用
- C. 不同计划包含不同能力，企业治理能力通常依赖更高层级
- D. 定价只和本地 Terraform CLI 版本有关

正确答案：C

解析：

A 太绝对。HCP Terraform 不是所有能力都永久免费。

B 也太绝对。不是所有功能都必须企业版才能用，基础平台能力可能在低层级计划中就能使用。

D 明显错误。HCP Terraform 定价和平台计划、资源规模、组织使用方式有关，不是由本地 CLI 版本决定。

C 最稳妥，因为它体现了考试最需要的判断：

> 不同计划，不同能力；企业治理能力通常属于更高层级。

---

### 题目 2

哪类能力通常更偏企业治理和合规场景？

- A. `terraform fmt`
- B. audit logging、drift detection、policy governance
- C. 本地变量默认值
- D. `terraform console`

正确答案：B

解析：

`terraform fmt` 是格式化命令。

本地变量默认值是 Terraform 配置语言的基础能力。

`terraform console` 是本地表达式调试工具。

只有 B 这一组关键词明显属于平台治理、审计、合规和企业协作场景。

考试看到：

- audit
- drift
- policy
- governance
- compliance
- approval
- team permission

就要立刻想到 HCP Terraform 平台治理能力。

---

### 题目 3

Airgap installation 更接近哪种场景？

- A. 无需联网或强隔离环境中的企业自托管部署
- B. 普通本地 Docker 练习
- C. `terraform validate` 的默认功能
- D. 每个免费 workspace 的必选设置

正确答案：A

解析：

airgap 关键词非常明显：强隔离、无公网、企业自托管。

它不是普通 Docker lab，也不是 Terraform CLI 校验命令，更不是免费 workspace 的默认设置。

## 8. 判断题讲解

### 判断题 1

复习 HCP Terraform 定价时，应把具体价格视为可能变化的信息。

答案：对

解析：

云产品和 SaaS 平台的价格、套餐名、功能边界都可能变化。考试更稳定的考法是能力分类和场景判断，而不是固定金额。

---

### 判断题 2

企业级审计、漂移检测和持续验证等功能通常是考试中需要识别的治理能力。

答案：对

解析：

这些都不是普通单机 Terraform CLI 的核心功能，而是平台化、组织化、合规化能力。

---

### 判断题 3

Terraform Enterprise 和托管的 HCP Terraform 在部署与运营责任上完全相同。

答案：错

解析：

HCP Terraform 是托管服务，平台维护责任主要在服务提供方。

Terraform Enterprise 是企业自托管，部署、运行、升级、维护责任更多在企业自己。

## 9. 场景题讲解

### 场景题 1

公司要求所有基础设施变更必须满足审计留痕、策略控制和漂移检测。你会把这些需求归类为普通 CLI 能力，还是 HCP Terraform 的平台治理能力？请说明原因。

参考答案：

这些需求应该归类为平台治理能力。

原因是：

- 审计留痕需要记录谁在什么时候做了什么操作
- 策略控制需要在组织层面强制执行规则
- 漂移检测需要持续发现真实资源和 Terraform 期望状态之间的差异
- 这些需求通常涉及团队协作、权限、审批、合规和自动化检测

Terraform CLI 可以执行 plan/apply，也可以在某次运行中发现差异，但它本身不是完整的企业治理平台。

更准确的表达是：

> CLI 负责执行 Terraform 工作流，HCP Terraform 或 Terraform Enterprise 负责把这个工作流平台化、审计化、治理化。

---

### 场景题 2

个人学习者没有付费计划，但课程讲到了 drift detection。面对这种情况，应该如何安排复习？

参考答案：

不需要为了考试强行购买付费能力。

复习时应该重点掌握：

- drift 是什么
- drift 为什么危险
- `terraform plan` 如何发现差异
- 平台级 drift detection 和手动 plan 的区别
- drift detection 为什么属于治理能力
- 哪些场景需要自动化漂移检测

个人学习者可以通过手动制造 drift 来理解概念：

1. 用 Terraform 创建资源
2. 手动去云控制台修改资源
3. 再运行 `terraform plan`
4. 观察 Terraform 如何发现真实资源与配置不一致

这样即使没有平台付费功能，也能理解 drift detection 的考试逻辑。

---

## 10. 一句话记忆法

> 免费账号能帮你入门平台概念，但企业治理能力不是“默认无限送”；看到 audit、policy、drift、continuous validation、airgap，就往高级治理或企业自托管方向判断。

更短一点：

> 价格别死背，能力要分类；CLI 管执行，平台管治理。

## 12. 最后复习提醒

本节不要卷价格表。

真正要练的是看到题目关键词之后，能迅速判断它属于哪一层：

1. 本地 Terraform CLI
2. HCP Terraform 基础平台能力
3. HCP Terraform 企业治理能力
4. Terraform Enterprise 自托管 / airgap 场景

考试一般不会奖励你背价格，但会奖励你不被“所有功能永久免费”“所有功能都必须企业版”“CLI 等于平台治理”这种说法带跑。
