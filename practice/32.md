## 2. 知识点讲解

Terraform provider 文档通常把内容分成两类：resource 用来创建和管理对象，data source 用来读取已有对象或环境信息。考试和生产排障都要求你能快速判断应该创建资源，还是只读取现有信息。

本实验使用 `data "aws_caller_identity"` 作为最小 AWS data source。它读取当前调用者的账号信息，不创建任何 AWS 资源。LocalStack 会返回模拟账号 `000000000000`，这可以帮助你验证 provider endpoint 是否正确指向 sandbox。

真实 AWS 中，这个 data source 常用于输出账号 ID、拼接 ARN、检查当前 profile 是否连到了预期账号。

## 3. 实验设计

背景：团队需要确认 Terraform 当前使用的是 sandbox 模拟账号，而不是真实 AWS 账号。

任务：读取 `aws_caller_identity` 和当前 region，并输出账号 ID、ARN、region。

预期结果：输出中的账号 ID 来自 LocalStack，且没有创建任何资源。

## 5. 运行手册

```sh
cd practice/labs/32
export AWS_ACCESS_KEY_ID=test
export AWS_SECRET_ACCESS_KEY=test
export AWS_DEFAULT_REGION=us-east-1
export LOCALSTACK_ENDPOINT=http://localhost:4566

bash scripts/check-sandbox.sh
terraform init -input=false
terraform fmt -check
terraform validate
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
bash scripts/verify.sh

```

## 7. 排障指南

1. 返回真实账号 ID：说明 provider 没有指向 LocalStack，立即停止。
2. STS endpoint 失败：检查 `LOCALSTACK_ENDPOINT` 是否为 `http://localhost:4566`。
3. 输出为空：检查是否声明了 `data "aws_caller_identity"`。
4. region 不符合预期：检查 `AWS_DEFAULT_REGION` 和变量默认值。
