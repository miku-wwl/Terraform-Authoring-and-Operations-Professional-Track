# Terraform 实操训练 64 讲解：`flatten` 与 `distinct`

## 1. 这一节到底在学什么？

Lab 64 学的是 Terraform 里两个非常实用的集合处理函数：

```hcl
flatten()
distinct()
```

它们经常和 `for` 表达式、嵌套类型、`for_each` 一起使用。

一句话理解：

> `flatten()` 负责把嵌套 list 拍平成一层；`distinct()` 负责从 list 中去掉重复值。

这两个函数看起来很小，但在真实 Terraform 模块里非常常见。

特别是你前面已经学了：

- Lab 59：for 表达式基础
- Lab 61：for 表达式进阶
- Lab 63：嵌套 for 表达式

Lab 64 就是继续补上嵌套 for 之后最常用的两个处理工具。

---

## 2. `flatten()` 是什么？

`flatten()` 用来把嵌套 list 展平成一层 list。

例如：

```hcl
locals {
  nested = [
    ["api", "worker"],
    ["web"]
  ]

  flat = flatten(local.nested)
}
```

结果是：

```hcl
[
  "api",
  "worker",
  "web"
]
```

也就是说：

```hcl
[
  ["api", "worker"],
  ["web"]
]
```

会变成：

```hcl
[
  "api",
  "worker",
  "web"
]
```

---

## 3. 为什么需要 `flatten()`？

因为嵌套 `for` 表达式经常会产生二维 list。

例如：

```hcl
locals {
  services = [
    {
      name  = "api"
      ports = [8080, 9090]
    },
    {
      name  = "web"
      ports = [3000]
    }
  ]

  service_ports_nested = [
    for svc in local.services : [
      for port in svc.ports : {
        service = svc.name
        port    = port
      }
    ]
  ]
}
```

这个结果是：

```hcl
[
  [
    {
      service = "api"
      port    = 8080
    },
    {
      service = "api"
      port    = 9090
    }
  ],
  [
    {
      service = "web"
      port    = 3000
    }
  ]
]
```

这是 `list(list(object))`。

但很多资源、output 或后续转换更想要一层 list：

```hcl
[
  {
    service = "api"
    port    = 8080
  },
  {
    service = "api"
    port    = 9090
  },
  {
    service = "web"
    port    = 3000
  }
]
```

这时就需要：

```hcl
flatten(...)
```

---

## 4. `flatten()` 的典型写法

嵌套 for + flatten 的经典模式：

```hcl
locals {
  service_ports = flatten([
    for svc in local.services : [
      for port in svc.ports : {
        service = svc.name
        port    = port
      }
    ]
  ])
}
```

这就是 Lab 63 的核心模式继续深化。

结构可以记成：

```hcl
flatten([
  for outer in outer_collection : [
    for inner in outer.inner_collection : {
      ...
    }
  ]
])
```

现实里常用于：

- 服务 × 端口
- 环境 × region
- 用户 × role
- team × repository
- subnet × route
- alert group × receiver
- cluster × namespace

---

## 5. `flatten()` 只展平 list

注意：`flatten()` 主要处理 list。

例如：

```hcl
flatten([
  ["a", "b"],
  ["c"]
])
```

可以。

但如果是 map 里面嵌套 map，它不会自动按你想象的方式处理。

例如：

```hcl
{
  api = {
    ports = [8080, 9090]
  }
}
```

这种结构要先用 `for` 表达式取出 list，再 flatten。

不要把 `flatten()` 理解成“所有嵌套结构一键拍平”。  
它不是万能结构清洗器。

---

## 6. `distinct()` 是什么？

`distinct()` 用来去掉 list 中的重复元素。

示例：

```hcl
locals {
  raw = ["api", "api", "worker", "web", "worker"]

  unique = distinct(local.raw)
}
```

结果是：

```hcl
[
  "api",
  "worker",
  "web"
]
```

它保留第一次出现的顺序，并去掉后续重复项。

一句话：

> `distinct()` 是 list 去重工具。

---

## 7. `distinct()` 和 `toset()` 的区别

`distinct()` 和 `toset()` 都能去重，但语义不同。

| 工具 | 返回类型 | 是否保留 list 语义 | 是否表达无序集合 |
|---|---|---|---|
| `distinct(list)` | list | 是 | 否 |
| `toset(list)` | set | 否 | 是 |

示例：

```hcl
distinct(["a", "a", "b"])
```

返回 list：

```hcl
["a", "b"]
```

而：

```hcl
toset(["a", "a", "b"])
```

返回 set：

```hcl
toset(["a", "b"])
```

如果你还需要 list，就用 `distinct()`。  
如果你要表达无序唯一集合，就用 `toset()`。

---

## 8. `flatten()` 和 `distinct()` 的组合

很常见的组合是：

```hcl
distinct(flatten(...))
```

例如你有多个服务，每个服务声明了依赖的 team：

```hcl
locals {
  services = [
    {
      name  = "api"
      teams = ["platform", "backend"]
    },
    {
      name  = "web"
      teams = ["frontend", "platform"]
    }
  ]

  all_teams = distinct(flatten([
    for svc in local.services : svc.teams
  ]))
}
```

中间结果：

```hcl
flatten([
  ["platform", "backend"],
  ["frontend", "platform"]
])
```

变成：

```hcl
["platform", "backend", "frontend", "platform"]
```

再 `distinct()`：

```hcl
["platform", "backend", "frontend"]
```

这就是：

> 先展开，再去重。

---

## 9. 为什么顺序是 `distinct(flatten(...))`？

通常应该先 `flatten()`，再 `distinct()`。

因为你要先把二维 list 变成一维 list，才能对所有元素整体去重。

例如：

```hcl
distinct(flatten([
  ["a", "b"],
  ["a", "c"]
]))
```

结果：

```hcl
["a", "b", "c"]
```

如果你只对内层分别 distinct，可能得不到全局去重效果。

所以常见顺序是：

```hcl
distinct(flatten(...))
```

而不是随便乱套。

## 13. 常见场景一：提取所有唯一端口

假设有多个服务：

```hcl

    {

      ports = [80, 443]
    },
    {
      name  = "web"

```

提取所有端口：



## 16. 常见场景四：展开对象后创建资源

`flatten()` 不只是提取简单值，也可以展开对象。

```hcl
locals {
  service_ports = flatten([
    for svc in local.services : [
      for port in svc.ports : {
        service = svc.name
        port    = port

    ]

  service_ports_by_key = {
    for item in local.service_ports :

```

```hcl

  filename = "${path.module}/output/${each.key}.txt"
  content  = jsonencode(each.value)
}
```

这就是 Lab 63 和 Lab 64 的组合。

## 17. `distinct()` 适合简单值，不适合复杂去重逻辑

`distinct()` 最适合对简单 list 去重：

```hcl
distinct(["a", "a", "b"])
```

对于对象 list：

```hcl
distinct([
  { name = "api", port = 8080 },
  { name = "api", port = 8080 }
])
```

Terraform 可以比较对象，但生产里复杂对象去重通常不建议依赖得太隐晦。

如果你真正关心的是某个字段唯一，比如 service name，最好明确提取字段：

```hcl
distinct([
  for svc in var.services : svc.name
])
```

或者转成 map：

```hcl
{
  for svc in var.services : svc.name => svc
}
```

这样意图更清楚。

---

## 18. `distinct()` 不等于校验唯一性

这个点很重要。

`distinct()` 会静默去重。

如果你的输入不应该重复，比如服务名必须唯一，用 `distinct()` 直接去重可能会掩盖数据问题。

例如：

```hcl
service_names = distinct([
  for svc in var.services : svc.name
])
```

如果两个服务都叫 `api`，它会变成一个 `api`。

但你真正应该做的是报错，而不是悄悄去重。

校验唯一性可以用测试或 check：

```hcl
length(local.service_names) == length(distinct(local.service_names))
```

如果长度不一致，说明有重复。

一句话：

> `distinct()` 是去重工具，不是数据质量免责工具。

---

## 19. 如何校验唯一性？

假设：

```hcl
locals {
  service_names = [
    for svc in var.services : svc.name
  ]

  unique_service_names = distinct(local.service_names)
}
```

可以检查：

```hcl
length(local.service_names) == length(local.unique_service_names)
```

如果相等，说明没有重复。  
如果不相等，说明有重复。

在 `terraform test` 中可以写断言：

```hcl
assert {
  condition     = length(local.service_names) == length(local.unique_service_names)
  error_message = "service names must be unique."
}
```

或者通过 output / check block 进行验证。

## 20. `flatten()` 的常见坑

### 坑 1：忘记 flatten

嵌套 for 后得到的是 list(list(...))，但你以为是一层 list。

结果后续资源或 output 类型不对。

---

### 坑 2：flatten 层级理解错

`flatten()` 会递归展平直接嵌套的 list，但不会把 map/object 里的 list 自动拿出来。

你要先用 for 把需要的 list 提取出来。

---

### 坑 3：对非 list 乱用

如果数据不是 list 嵌套结构，`flatten()` 不是正确工具。

先看清楚类型，再用函数。

---

## 21. `distinct()` 的常见坑

### 坑 1：以为 distinct 会排序

不会。

`distinct()` 去重，但不是排序。

如果要排序：

```hcl
sort(distinct(local.names))
```

---

### 坑 2：用 distinct 掩盖重复数据

如果重复是错误，就应该校验失败，而不是静默去重。

---

### 坑 3：复杂对象去重语义不清楚

复杂对象去重最好先明确你到底按什么字段去重。

---

## 22. `flatten`、`distinct`、`sort` 的组合

有时为了输出稳定，你会看到：

```hcl
sort(distinct(flatten([
  for svc in var.services : svc.teams
])))
```

含义是：

1. `for svc in var.services : svc.teams`  
   提取每个服务的 team list。

2. `flatten(...)`  
   把二维 list 变成一维。

3. `distinct(...)`  
   去重。

4. `sort(...)`  
   排序，让输出稳定。

这在 output 和 test 中很实用。

如果只用 `distinct()`，结果顺序依赖原始输入顺序。  
如果需要稳定展示，可以加 `sort()`。

---

## 23. 和 `set` 的关系

Lab 56 学过 set 类型。  
set 本身就是无序去重集合。

如果你只是表达唯一成员集合，可以用：

```hcl
toset(local.names)
```

如果你需要保留 list 并可能排序输出，可以用：

```hcl
sort(distinct(local.names))
```

选择方式：

| 目标 | 推荐 |
|---|---|
| 表达无序唯一集合 | `toset()` |
| 保留 list 类型并去重 | `distinct()` |
| 稳定输出唯一 list | `sort(distinct(...))` |

## 28. 面试/考试记忆版

可以这样记：

- `flatten()` 把嵌套 list 展平成一层 list。
- 嵌套 for 表达式常常需要 `flatten()`。
- `distinct()` 对 list 去重。
- `distinct()` 不排序。
- 稳定唯一输出常用 `sort(distinct(...))`。
- 展平并去重常用 `distinct(flatten(...))`。
- `distinct()` 会静默去重，不能代替唯一性校验。
- 唯一性校验可以比较 `length(list)` 和 `length(distinct(list))`。
- set 表达无序唯一集合，distinct 保留 list 语义。
- 展平对象后创建资源，通常还要转成 map 给 `for_each`。

一句话总结：

> `flatten()` 解决嵌套层级，`distinct()` 解决重复元素；二者常和 `for` 表达式一起，把复杂输入清洗成 Terraform 可用数据。
