# Terraform 实操训练 45：资源行为与 meta-argument

## 1. 课程元数据

- 课程编号：45
- 来源文件：`doc/45.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：资源行为、创建、销毁、更新、替换、meta-argument
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Terraform 会根据配置、state 和远端对象推导操作：创建缺失对象、销毁已移除对象、原地更新可变属性、替换不可原地更新的对象。Meta-argument 用于调整这些默认行为。

## 3. 实验设计

背景：团队需要复盘 Terraform 如何决定资源动作。

问题：不了解默认行为会误判 plan。

任务：记录四类资源行为和常见 meta-argument。

预期结果：输出包含 `lifecycle`。

## 4. 实验文件

- `practice/labs/45/main.tf`
- `practice/labs/45/tests/meta_argument_basics.tftest.hcl`
- `practice/labs/45/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/45
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。完整流程通过，测试确认四类行为和 lifecycle 已记录。

## 7. 排障指南

1. plan 动作不理解：先区分 create/update/replace/destroy。
2. 替换风险高：检查哪些属性是 ForceNew。
3. 不要用 lifecycle 掩盖建模问题。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
