# 第 142 节：AWS CLI source_profile 配置

## 知识点讲解

本节对应 `doc/142.txt` 的课程录音转写。主题是 `AWS CLI source_profile 配置`，重点不是把真实 AWS 账号作为练习环境，而是在本地 LocalStack 中理解 Terraform AWS Provider 如何声明、查询、绑定和销毁 AWS 资源。

LocalStack 使用 `http://localhost:4566` 作为 AWS API endpoint，凭证固定为 `test/test`。这可以训练 Terraform 写法、依赖关系、状态变化和输出设计，但不能替代真实 AWS 的 IAM 权限评估、真实扩缩容或真实网络连通性。

## 实验目标

- 创建 AWS CLI base profile。
- 创建通过 source_profile 继承凭证的 lab profile。
- 理解 source_profile 与 assume role 配置的关系。

## 实验边界

- 不访问真实 AWS 账号。
- 不写入真实 access key、secret key 或 root 账号信息。
- 使用 Terraform AWS Provider + LocalStack 进行练习。
- 本节包含 LocalStack Community 无法完全复刻的真实 AWS 行为，因此验证重点是 Terraform 语法、Provider 调用和状态闭环。
