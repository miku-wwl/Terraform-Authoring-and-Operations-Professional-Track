# Terraform 实操训练 18：`TF_IN_AUTOMATION` 与自动化日志

## 1. 课程元数据

- 课程编号：18
- 来源文件：`doc/18.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`TF_IN_AUTOMATION`、CI/CD 日志、减少下一步提示
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Terraform 默认会输出一些适合人工阅读的下一步提示，例如 init 后提示运行 plan。CI/CD 中这些提示通常没有价值，只会增加日志噪音。

设置 `TF_IN_AUTOMATION=true` 后，Terraform 会减少这类面向人工的提示，让日志更短、更适合排障和长期归档。

## 3. 实验设计

### 背景

平台团队要精简 Terraform 流水线日志。

### 问题

默认输出包含大量下一步建议，影响排障阅读。

### 任务

设置 `TF_IN_AUTOMATION=true`，生成自动化日志策略文件。

### 预期结果

`init/fmt/validate/test/plan/apply/destroy` 通过，且 init 输出比默认模式更简洁。

## 5. 运行手册

```sh
cd /workspace/practice/labs/18
export TF_IN_AUTOMATION=true
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

关键结果：

- `terraform init` 在 `TF_IN_AUTOMATION=true` 下输出更简洁。
- `terraform test` 返回 `1 passed, 0 failed`。
- `plan/apply/destroy` 全链路通过。

## 7. 排障指南

1. 日志仍然很长：确认环境变量在当前 shell 中生效。
2. PowerShell 设置方式不同：使用 `$env:TF_IN_AUTOMATION='true'`。
3. Linux/macOS 设置方式：使用 `export TF_IN_AUTOMATION=true`。

## 8. 清理方式

```sh
terraform destroy -auto-approve
unset TF_IN_AUTOMATION
```
