## 2. 知识点讲解

HCP Terraform 的权限可以在 organization、project、workspace 等层面配置。课程重点讲到 team access：你可以为某个 team 授予 workspace 访问权限，并选择 read、plan、write、admin 或 custom 等级。更细粒度的 custom 权限还可以控制变量、state、run task、Sentinel mocks 等能力。

权限题的核心是最小权限原则。开发者可能只需要 plan 或 apply 某个 workspace，不应该默认拥有组织管理员权限。安全团队可能需要读取 plan 和 policy 结果，但不一定能改变量或执行 apply。审计人员可能只需要读取输出和 run 历史，不应读取敏感变量。

考试里常用场景题考你能否区分：读取输出、读取完整 state、修改变量、执行 apply、管理 workspace、管理 organization 不是同一类权限。尤其是 state 权限很敏感，因为 state 可能包含资源属性和敏感信息。

## 3. 考试选择题

1. 哪项最符合 HCP Terraform 权限管理的最佳实践？
   - A. 按 team 和职责授予最小必要权限
   - B. 所有人加入 owners team
   - C. 把所有密钥写入 Git
   - D. 禁用所有 workspace

2. Workspace 权限中，custom 权限的价值是什么？
   - A. 对变量、state、run 等能力做更细粒度控制
   - B. 自动创建云资源
   - C. 替代 Terraform module
   - D. 生成 provider 文档

3. 为什么 state 访问权限需要谨慎控制？
   - A. State 可能包含资源属性和敏感信息
   - B. State 只包含空白文本
   - C. State 只保存 Markdown
   - D. State 与基础设施无关

## 4. 判断题

1. HCP Terraform 可以在 workspace 级别为 team 配置访问权限。
2. 最小权限原则意味着普通开发者默认应该拥有 organization owner 权限。
3. 读取 outputs only 和读取完整 state 是不同权限边界。

## 5. 场景题

1. 开发团队需要运行 plan，但不能 apply 生产 workspace。你会如何配置权限方向？

2. 审计团队需要查看运行记录和输出，但不应看到敏感 state 细节。应该避免授予哪类权限？

## 6. 参考答案与解析

选择题：1-A，2-A，3-A。

判断题：1-对，2-错，3-对。

场景题参考：

1. 为开发 team 授予生产 workspace 的 read/plan 类权限，避免授予 write、apply 或 admin 级别。
2. 避免授予完整 state read/write 权限。可以根据需要提供 outputs only 或只读 run 历史权限。

## 7. 复习提醒

- 权限题优先套最小权限原则。
- State 权限非常敏感，不要默认放开。
- Custom 权限是细粒度治理的关键词。
