# Terraform 实操训练 28：Indented heredoc 的前导空白规则

## 1. 课程元数据

- 课程编号：28
- 来源文件：`doc/28.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：leading whitespace、共同缩进、相对缩进
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Indented heredoc 会查找所有行共同的最短前导空白，并从每一行移除同样数量。它不会把所有缩进都清零，而是保留行与行之间的相对缩进。

理解这个规则可以避免生成 YAML、脚本或配置文件时出现缩进错误。

## 3. 实验设计

背景：平台团队要生成 YAML 配置片段。

问题：Terraform 代码中为了可读性缩进了 heredoc，但生成文件不能带外层缩进。

任务：使用 `<<-EOT` 生成 YAML，验证第一行无外层缩进、第二行保留相对缩进。

预期结果：输出三行 YAML，`name` 行保留两个空格。

状态：通过。

已执行 `init/fmt/validate/test/plan/apply/output/destroy`。`terraform test` 返回 `1 passed, 0 failed`，输出第一行为 `app:`，第二行为带两个空格的 `name: payments`，证明相对缩进被保留。

## 7. 排障指南

1. 第一行仍有空格：检查是否所有行都有共同前导空白。
2. 相对缩进丢失：不要让所有行缩进完全相同后再期望保留层级。
3. 空行影响判断：空行可能影响阅读，必要时使用 `trimspace`。
4. 文件格式错误：用目标格式工具验证生成内容。
