# Terraform 实操训练 22：`random_integer` 与唯一命名

## 1. 课程元数据

- 课程编号：22
- 来源文件：`doc/22.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`random_integer`、唯一后缀、state 中的随机值
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

`random_integer` 会在指定范围内生成随机整数，常用于测试环境、临时环境或需要降低命名冲突概率的场景。随机资源的结果会进入 Terraform state，因此后续 apply 不会无故改变。

如果要重新生成随机值，需要 taint、replace 或修改触发替换的参数。生产中要注意随机值是否会影响资源不可变名称。

## 3. 实验设计

背景：自动化流水线需要创建带唯一后缀的产物名。

问题：固定名称会冲突。

任务：使用 `random_integer` 生成本地 artifact 文件名。

预期结果：随机后缀在 1000 到 9999 之间，文件名包含该后缀。

## 5. 运行手册

```sh
cd /workspace/practice/labs/22
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

已执行 `init/fmt/validate/test/plan/apply/output/destroy`。`terraform test` 返回 `1 passed, 0 failed`，随机后缀落在 `1000` 到 `9999` 范围内，并被用于本地 artifact 文件名。

## 7. 排障指南

1. 随机值没变化：这是 state 保持稳定的正常行为。
2. 想重新生成：使用 `terraform apply -replace=random_integer.suffix`。
3. 名称仍冲突：扩大随机范围或增加环境前缀。
4. 删除 state 后值变化：state 是随机值稳定性的来源。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf output .terraform .terraform.lock.hcl
```
