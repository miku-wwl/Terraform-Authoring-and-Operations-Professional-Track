# Terraform 实操训练 7：`-input=false` 与非交互式失败

## 1. 课程元数据

- 课程编号：7
- 来源文件：`doc/7.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`-input=false`、必填变量、CI/CD 快速失败
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Terraform 在缺少必填变量时，默认可能提示用户输入。这在人工终端里可以接受，但在 CI/CD 中会让任务挂起。`-input=false` 的意义是让自动化任务不要等待人工输入，而是在缺少值时立即失败。

这个要求不只适用于 `plan`，也适用于 `init/apply/destroy` 等生命周期命令。只要配置依赖输入变量，自动化脚本就必须显式传入变量或变量文件。

## 3. 实验设计

### 背景

流水线要生成一个构建产物文件名，但文件名必须由流水线参数提供。

### 问题

如果没有显式提供变量，Terraform 会要求输入，自动化任务会卡住。

### 任务

用 `-input=false` 和 `-var artifact_name=ci-artifact.txt` 完成 plan/apply/destroy。

### 预期结果

生成 `output/ci-artifact.txt`，并且测试确认命令链包含 `-input=false`。

## 5. 运行手册

```sh
cd /workspace/practice/labs/7
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan -var artifact_name=ci-artifact.txt
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve -var artifact_name=ci-artifact.txt
```

## 6. 验证报告

状态：通过。

关键结果：

- `terraform test` 返回 `1 passed, 0 failed`。
- `plan/apply` 创建 1 个本地文件资源。
- 首次验证暴露了 `destroy` 同样需要变量的问题，已修正运行手册。
- 修正后 `destroy -var artifact_name=ci-artifact.txt` 成功清理资源。

## 7. 排障指南

1. 出现 `No value for required variable`：给 `plan` 或 `destroy` 补 `-var`。
2. 自动化任务卡住：确认使用了 `-input=false`。
3. 文件名校验失败：`artifact_name` 必须是小写 txt 文件名。

## 8. 清理方式

```sh
terraform destroy -auto-approve -var artifact_name=ci-artifact.txt
```
