# Terraform 实操训练 59 讲解：for 表达式基础

## 1. 这一节到底在学什么？

Lab 59 学的是 Terraform 里的 **for 表达式**。

前面几节你已经学过：

- `list`：有序集合
- `map`：key-value 集合
- `object`：结构化对象
- 嵌套类型：复杂配置建模
- `for_each`：根据 map/set 创建多个资源实例
- 条件表达式：根据条件选择值

这一节的 `for` 表达式，重点是**转换数据**。

一句话理解：

> Terraform `for` 表达式是用来遍历集合并生成新集合的工具。

它不是用来“循环执行命令”，而是用来把一个数据结构变成另一个数据结构。

---

## 2. `for` 表达式和 `for_each` 的区别

这两个名字很像，但作用不同。

| 工具 | 作用 | 典型用途 |
|---|---|---|
| `for` 表达式 | 转换数据 | 把 list/map 变成新的 list/map |
| `for_each` | 创建多个资源实例 | 根据 map/set 创建多个 resource/module |

简单说：

```hcl
for
```

是表达式，用来生成值。

```hcl
for_each
```

是 meta-argument，用来创建多个资源实例。

例如：

```hcl
locals {
  upper_names = [
    for name in var.names : upper(name)
  ]
}
```

这是 `for` 表达式。

而：

```hcl
resource "local_file" "example" {
  for_each = var.files
}
```

这是 `for_each`。

一句话：

> `for` 管数据转换，`for_each` 管资源实例。

---

## 3. list 到 list 的转换

最基础的 `for` 表达式是 list 转 list。

示例：

```hcl
locals {
  names = ["api", "worker", "frontend"]

  upper_names = [
    for name in local.names : upper(name)
  ]
}
```

结果：

```hcl
[
  "API",
  "WORKER",
  "FRONTEND"
]
```

结构拆解：

```hcl
[
  for name in local.names : upper(name)
]
```

含义是：

1. 遍历 `local.names`。
2. 每次取出一个元素，临时叫 `name`。
3. 对 `name` 执行 `upper(name)`。
4. 把结果组成新的 list。

---

## 4. list 到 map 的转换

`for` 表达式也可以把 list 转成 map。

示例：

```hcl
locals {
  names = ["api", "worker", "frontend"]

  name_map = {
    for name in local.names : name => upper(name)
  }
}
```

结果：

```hcl
{
  api      = "API"
  worker   = "WORKER"
  frontend = "FRONTEND"
}
```

这里的关键是：

```hcl
name => upper(name)
```

左边是 map key。  
右边是 map value。

这种写法非常实用，尤其是把 list(object) 转换成 map(object)，然后给 `for_each` 使用。

---

## 5. map 到 list 的转换

假设有一个 map：

```hcl
locals {
  service_ports = {
    api    = 8080
    worker = 9090
    web    = 3000
  }
}
```

可以生成 list：

```hcl
locals {
  service_summaries = [
    for name, port in local.service_ports : "${name}:${port}"
  ]
}
```

结果类似：

```hcl
[
  "api:8080",
  "web:3000",
  "worker:9090"
]
```

注意：map 本身不是用来表达业务顺序的。  
如果输出顺序很重要，要考虑排序。

---

## 6. map 到 map 的转换

也可以从一个 map 生成另一个 map。

```hcl
locals {
  service_ports = {
    api    = 8080
    worker = 9090
  }

  service_urls = {
    for name, port in local.service_ports :
    name => "http://${name}:${port}"
  }
}
```

结果：

```hcl
{
  api    = "http://api:8080"
  worker = "http://worker:9090"
}
```

这在生成配置、输出摘要、模块输入转换时非常常见。

---

## 7. `for` 表达式的两种变量写法

遍历 list 时，通常写一个变量：

```hcl
[for item in var.items : item]
```

也可以写两个变量：

```hcl
[for index, item in var.items : "${index}-${item}"]
```

例如：

```hcl
locals {
  names = ["api", "worker"]

  indexed_names = [
    for idx, name in local.names : "${idx}-${name}"
  ]
}
```

结果：

```hcl
[
  "0-api",
  "1-worker"
]
```

遍历 map 时，常用两个变量：

```hcl
[for key, value in var.map : "${key}=${value}"]
```

这样能同时拿到 key 和 value。

---

## 8. `for` 表达式里的过滤：`if`

`for` 表达式可以加 `if` 条件过滤元素。

示例：

```hcl
locals {
  names = ["api", "worker", "frontend"]

  api_only = [
    for name in local.names : name
    if name == "api"
  ]
}
```

结果：

```hcl
[
  "api"
]
```

再比如过滤端口：

```hcl
locals {
  ports = [80, 443, 8080, 9090]

  high_ports = [
    for port in local.ports : port
    if port > 1024
  ]
}
```

结果：

```hcl
[
  8080,
  9090
]
```

一句话：

> `for ... : ... if ...` 可以边遍历边筛选。

---

## 9. `for` 表达式和条件表达式可以组合

Lab 58 学了条件表达式。  
`for` 表达式里也可以用条件表达式。

示例：

```hcl
locals {
  environments = ["dev", "prod"]

  sizes = [
    for env in local.environments : env == "prod" ? "large" : "small"
  ]
}
```

结果：

```hcl
[
  "small",
  "large"
]
```

也可以生成 map：

```hcl
locals {
  env_sizes = {
    for env in local.environments :
    env => env == "prod" ? "large" : "small"
  }
}
```

结果：

```hcl
{
  dev  = "small"
  prod = "large"
}
```

这就是 `for` + 条件表达式的组合。

## 10. list(object) 转 map(object)：生产高频写法

这是非常重要的生产用法。

假设输入是：

```hcl
variable "services" {
  type = list(object({
    name = string
    port = number
  }))

  default = [
    {
      name = "api"
      port = 8080
    },
    {
      name = "worker"
      port = 9090
    }
  ]
}
```

如果直接用 list + count，可能有索引漂移风险。

可以先转换成 map：

```hcl
locals {
  services_by_name = {
    for svc in var.services : svc.name => svc
  }
}
```

得到：

```hcl
{
  api = {
    name = "api"
    port = 8080
  }

  worker = {
    name = "worker"
    port = 9090
  }
}
```

然后给 `for_each`：

```hcl
resource "local_file" "service" {
  for_each = local.services_by_name

  filename = "${path.module}/output/${each.key}.txt"
  content  = "port=${each.value.port}"
}
```

这就是很典型的 Terraform 工程化写法。

---

## 11. 为什么 list(object) 转 map(object) 很重要？

因为 Terraform 资源实例需要稳定身份。

list(object) 的身份通常靠位置：

```text
[0]
[1]
[2]
```

map(object) 的身份靠 key：

```text
["api"]
["worker"]
["frontend"]
```

如果你要用这些对象创建长期资源，map key 通常更稳定。

所以 `for` 表达式经常承担一个角色：

> 把用户友好的输入结构转换成 Terraform 更适合管理 state 的结构。

这就是它的生产价值。

## 12. for 表达式和输出

`for` 表达式经常用于 output。

例如有多个资源：

```hcl
resource "local_file" "service" {
  for_each = var.services
}
```

输出所有文件名：

```hcl
output "service_files" {
  value = {
    for name, file in local_file.service :
    name => file.filename
  }
}
```

这样 output 会是一个 map：

```hcl
{
  api    = ".../api.txt"
  worker = ".../worker.txt"
}
```

这比输出一个无意义 list 更容易读。

---

## 13. for 表达式和 tags

生成 tags 也很常见。

例如统一把 tag value 加前缀：

```hcl
locals {
  raw_tags = {
    Owner = "platform"
    Env   = "dev"
  }

  normalized_tags = {
    for key, value in local.raw_tags :
    key => lower(value)
  }
}
```

结果：

```hcl
{
  Owner = "platform"
  Env   = "dev"
}
```

也可以过滤掉空值：

```hcl
locals {
  tags = {
    for key, value in var.tags :
    key => value
    if value != null && value != ""
  }
}
```

这在真实模块里非常实用。

## 14. for 表达式和嵌套类型

Lab 53、54 学过嵌套类型和读取嵌套值。  
`for` 表达式可以处理这些复杂结构。

例如：

```hcl
locals {
  services = {
    api = {
      ports = [
        {
          name = "http"
          port = 8080
        },
        {
          name = "metrics"
          port = 9090
        }
      ]
    }
  }

  api_ports = [
    for p in local.services["api"].ports : p.port
  ]
}
```

结果：

```hcl
[
  8080,
  9090
]
```

这就是从嵌套对象里提取字段。

## 18. 常见坑

### 坑 1：list 和 map 输出符号搞混

生成 list 用方括号：

```hcl
[for item in var.items : item]
```

生成 map 用大括号：

```hcl
{ for item in var.items : item.name => item }
```

如果你要生成 map，必须提供 key：

```hcl
key => value
```

---

### 坑 2：map key 重复

例如：

```hcl
{
  for svc in var.services : svc.name => svc
}
```

如果两个服务都叫：

```text
api
```

就会 key 冲突。

所以转换成 map 时，key 必须唯一。

如果 key 不唯一，要先修输入数据模型，别硬绕。

---

### 坑 3：过滤条件写错

例如：

```hcl
[for port in var.ports : port if port > 1024]
```

如果你本来想保留 80 和 443，却写成 `port > 1024`，结果会反。

这类逻辑要用 `terraform test` 固定。

---

### 坑 4：过度写复杂 for 表达式

Terraform 支持复杂表达式，但不是越炫越好。

如果一行里又嵌套 for，又嵌套条件，又嵌套函数，可读性会很差。

可以拆成 locals：

```hcl
locals {
  services_by_name = {
    for svc in var.services : svc.name => svc
  }

  enabled_services = {
    for name, svc in local.services_by_name :
    name => svc
    if svc.enabled
  }
}
```

比一行写完更容易维护。

---

### 坑 5：把 for 表达式当循环命令

Terraform 的 `for` 表达式不是程序语言里的循环执行。

它不会“逐个执行命令”。  
它只是生成一个新值。

如果你想创建多个资源，应该把生成的数据交给 `for_each`。

---

## 19. for 表达式和 locals

`for` 表达式很适合放在 `locals` 里做中间转换。

例如：

```hcl
locals {
  services_by_name = {
    for svc in var.services : svc.name => svc
  }

  enabled_services = {
    for name, svc in local.services_by_name :
    name => svc
    if svc.enabled
  }
}
```

然后资源里使用：

```hcl
for_each = local.enabled_services
```

这样资源块更干净。

好的 Terraform 模块通常会用 locals 做数据整形，再把整形后的数据交给资源。

---

## 20. for 表达式和 `flatten()`

有时 `for` 表达式会产生嵌套 list。

例如：

```hcl
locals {
  nested = [
    for svc in var.services : [
      for port in svc.ports : {
        service = svc.name
        port    = port
      }
    ]
  ]
}
```

结果是 list 里套 list。

这时可以用：

```hcl
flatten(local.nested)
```

变成一层 list。

完整写法：

```hcl
locals {
  service_ports = flatten([
    for svc in var.services : [
      for port in svc.ports : {
        service = svc.name
        port    = port
      }
    ]
  ])
}
```

这在处理“服务 × 端口”、“环境 × region”这种组合时很常见。

---

## 21. for 表达式和 `toset()`

如果你要去重，可以配合 `toset()`。

```hcl
locals {
  unique_envs = toset([
    for svc in var.services : svc.environment
  ])
}
```

这会提取所有服务的 environment，并去重。

如果只是展示，可以再排序：

```hcl
sort(tolist(local.unique_envs))
```

这和 Lab 56 的 set 类型可以串起来。

## 22. for 表达式和类型稳定

`for` 表达式生成的结果类型要清楚。

例如：

```hcl
[for port in var.ports : tostring(port)]
```

生成 list(string)。

```hcl
[for port in var.ports : port]
```

如果 `var.ports` 是 list(number)，生成 list(number)。

复杂模块里，类型不清楚会导致后续资源参数报错。  
所以要有意识地控制输出类型。

## 26. SRE / Platform Engineering 视角

从平台工程角度看，`for` 表达式是模块内部的数据整形工具。

用户传进来的数据，不一定就是资源最想要的形态。

例如用户为了可读性传：

```hcl
services = [
  {
    name = "api"
    port = 8080
  },
  {
    name = "worker"
    port = 9090
  }
]
```

但资源为了 state 稳定，可能需要：

```hcl
{
  api = {
    name = "api"
    port = 8080
  }
  worker = {
    name = "worker"
    port = 9090
  }
}
```

这个转换就靠 `for` 表达式。

所以成熟模块通常分三步：

1. 输入变量：对用户友好。
2. locals 整形：对 Terraform state 和资源创建友好。
3. resource/module：使用整形后的数据。

这就是平台工程味儿。

## 27. 面试/考试记忆版

可以这样记：

- `for` 表达式用来转换集合。
- 生成 list 用 `[]`。
- 生成 map 用 `{}` 和 `key => value`。
- 可以遍历 list、set、map。
- 可以加 `if` 过滤元素。
- 可以配合条件表达式转换值。
- 常用于把 `list(object)` 转成 `map(object)`。
- 常用于为 `for_each` 准备输入。
- map key 必须唯一。
- 复杂 for 表达式建议拆到 locals。
- `for` 是生成值，不是创建资源；创建资源用 `for_each`。

一句话总结：

> Terraform for 表达式的核心价值，是把输入数据转换成更适合模块、资源和 state 管理的结构。
