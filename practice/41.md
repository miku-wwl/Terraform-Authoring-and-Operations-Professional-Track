# Terraform 实操训练 41：Check block

## 1. 课程元数据

- 课程编号：41
- 来源文件：`doc/41.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：check block、资源外检查、合约验证
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

`check` block 不绑定单个资源生命周期，适合表达全局健康检查、外部依赖检查或合规合约。它和 variable validation、precondition 的定位不同，更像一组旁路验证。

## 3. 实验设计

背景：服务地址必须使用 HTTPS。

问题：HTTP 地址不符合生产安全要求。

任务：用 `check` block 校验 URL 前缀。

预期结果：默认 HTTPS 地址通过。

## 4. 实验文件

- `practice/labs/41/main.tf`
- `practice/labs/41/tests/check_block.tftest.hcl`
- `practice/labs/41/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/41
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。完整流程通过，测试确认服务 URL 使用 HTTPS。

## 7. 排障指南

1. check 失败：检查变量值是否符合合约。
2. check 不应替代资源依赖：真实依赖仍要明确建模。
3. 检查外部服务时要考虑网络不稳定。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
