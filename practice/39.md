# Terraform 实操训练 39：Precondition 与 postcondition

## 1. 课程元数据

- 课程编号：39
- 来源文件：`doc/39.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：precondition、postcondition、资源级约束
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Precondition 在资源评估前检查条件，适合阻止不合规请求；postcondition 在对象评估后检查结果，适合确认远端或计算结果满足约束。二者都写在 `lifecycle` 中。

## 3. 实验设计

背景：开发环境只允许小规格计算请求。

问题：大规格会带来成本风险。

任务：在资源中添加前置和后置条件。

预期结果：默认 small 通过，xlarge 被拒绝。

## 4. 实验文件

- `practice/labs/39/main.tf`
- `practice/labs/39/tests/pre_post_conditions.tftest.hcl`

## 5. 运行手册

```sh
cd /workspace/practice/labs/39
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。完整流程通过，测试确认 precondition 和 postcondition 均可通过。

## 7. 排障指南

1. precondition 失败：检查输入是否符合策略。
2. postcondition 失败：检查资源计算结果或远端返回值。
3. 条件过度复杂：拆成局部变量提高可读性。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
