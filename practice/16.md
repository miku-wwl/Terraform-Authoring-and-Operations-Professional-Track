# Terraform 实操训练 16：保存 Plan 到文件

## 1. 课程元数据

- 课程编号：16
- 来源文件：`doc/16.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`terraform plan -out`、二进制 plan、`terraform show`、审批一致性
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

保存 plan 的核心价值是让审批内容和执行内容一致。`terraform plan -out=tfplan` 会生成二进制 plan 文件，后续 `terraform apply tfplan` 会执行当时计划中的动作，即使源码后来被改动也不会改变这次 apply。

需要审查时，可以用 `terraform show tfplan` 看文本，也可以用 `terraform show -json tfplan` 输出 JSON 给工具分析。

## 3. 实验设计

### 背景

生产变更需要先提交 plan 供审批。

### 问题

如果审批后重新运行裸 `apply`，实际执行内容可能和审批内容不一致。

### 任务

保存 plan，查看 plan 文本和 JSON，再从同一个 plan apply。

### 预期结果

生成 `output/approved-change.txt`，并验证命令链包含 `terraform apply -auto-approve tfplan`。

## 5. 运行手册

```sh
cd /workspace/practice/labs/16
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform show tfplan
terraform show -json tfplan > plan.json
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

关键结果：

- `terraform test` 返回 `1 passed, 0 failed`。
- `terraform plan` 保存到 `tfplan`。
- `terraform show tfplan` 可读。
- `terraform show -json tfplan > plan.json` 成功生成 JSON。
- apply 和 destroy 成功。

## 7. 排障指南

1. 直接打开 `tfplan` 乱码：它是二进制文件。
2. 审批后代码又改了：仍使用原来的 `tfplan` apply。
3. 工具需要解析 plan：先转成 JSON。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan plan.json
```
