# Terraform 实操训练 23：Targeting 与依赖方向

## 1. 课程元数据

- 课程编号：23
- 来源文件：`doc/23.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`-target`、上游依赖、下游资源、完整收敛
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

targeting 会包含目标资源所依赖的上游资源，但不会自动包含依赖目标资源的下游资源。这个方向性非常关键：如果你 target 一个基础资源，依赖它的业务资源可能不会被创建或更新。

因此 targeted apply 后通常还要跑一次完整 plan，确认整体配置是否已经收敛。

## 3. 实验设计

背景：manifest 文件名依赖随机 build number。

问题：只 target 随机数不会自动创建依赖它的 manifest。

任务：先 target `random_integer.build_number`，再执行完整 plan/apply。

预期结果：理解 target 的依赖方向，并最终完成完整收敛。

## 4. 实验文件

- `practice/labs/23/main.tf`
- `practice/labs/23/tests/target_dependencies.tftest.hcl`
- `practice/labs/23/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/23
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -target=random_integer.build_number -out=tfplan
terraform apply -auto-approve tfplan
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

已执行 `init/fmt/validate/test/plan/apply/output/destroy`，`terraform test` 返回 `1 passed, 0 failed`。额外验证 targeted workflow：先 target `random_integer.build_number` 时不会生成 manifest；再执行完整 plan/apply 后生成 `manifest-*.txt`。首次验证发现 manifest 文件名在 plan 阶段未知，已将测试改为 `command = apply`。

## 7. 排障指南

1. target 后文件没生成：这是预期现象，下游未被包含。
2. 出现警告：targeting 警告说明结果可能不完整。
3. 完整 plan 仍有变更：继续 apply 直到收敛。
4. 依赖不明显：用引用表达式或 `terraform graph` 辅助分析。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf output .terraform .terraform.lock.hcl
```
