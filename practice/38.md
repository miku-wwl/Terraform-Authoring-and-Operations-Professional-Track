# Terraform 实操训练 38：多条件变量验证

## 1. 课程元数据

- 课程编号：38
- 来源文件：`doc/38.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`contains()`、允许列表、环境约束
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

`contains()` 常用于白名单验证。生产中可以限制实例规格、环境名称、区域代号或模块模式，避免用户传入未批准的配置。

## 3. 实验设计

背景：开发环境只允许使用批准过的规格和环境名。

问题：随意传值会造成成本和治理风险。

任务：为 `instance_size` 和 `environment` 增加允许列表。

预期结果：默认值通过，非法规格被拒绝。

## 4. 实验文件

- `practice/labs/38/main.tf`
- `practice/labs/38/tests/advanced_validation.tftest.hcl`
- `practice/labs/38/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/38
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。完整流程通过，测试确认规格来自允许列表。

## 7. 排障指南

1. 新规格无法使用：先加入允许列表。
2. 大小写错误：允许列表是精确匹配。
3. 错误信息要包含可选值，方便修复。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
