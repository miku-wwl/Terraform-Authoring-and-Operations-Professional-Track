# Terraform 实操训练 49：`ignore_changes`

## 1. 课程元数据

- 课程编号：49
- 来源文件：`doc/49.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：漂移、外部修改、`ignore_changes`
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

`ignore_changes` 允许 Terraform 忽略指定属性的外部变化。它适合接纳由其他系统管理的标签、注释或运行时字段，但不能滥用，否则会掩盖真实漂移。

## 3. 实验设计

背景：文件内容可能被外部系统改写。

问题：Terraform 不应把该内容恢复为基线。

任务：对 `local_file.managed_note.content` 设置 `ignore_changes`。

预期结果：外部修改后 plan 不尝试恢复 content。

## 4. 实验文件

- `practice/labs/49/main.tf`
- `practice/labs/49/tests/ignore_changes.tftest.hcl`
- `practice/labs/49/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/49
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
printf '外部系统写入的内容\n' > output/managed-note.txt
terraform plan -input=false -no-color
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。完整流程通过，测试确认忽略属性为 `content`。

## 7. 排障指南

1. 仍出现漂移：确认属性路径写对。
2. 忽略过多：只忽略确实由外部系统管理的字段。
3. 文件不存在：先完成 apply。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf output .terraform .terraform.lock.hcl
```
