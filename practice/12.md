# Terraform 实操训练 12：显式 Provider Installation

## 1. 课程元数据

- 课程编号：12
- 来源文件：`doc/12.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`provider_installation`、`filesystem_mirror`、`direct` 回退
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

显式 provider installation 可以精确控制 Terraform 从哪里安装 provider。`filesystem_mirror` 适合离线、内网和受控环境；`direct` 则允许 Terraform 回到 origin registry 下载。

生产上要清楚边界：真正 air-gapped 环境不应依赖 `direct` 回退，否则一旦 mirror 缺包，初始化仍会失败。

## 3. 实验设计

### 背景

团队既有离线 runner，也有允许联网的开发环境。

### 问题

不同环境对 provider 安装源要求不同。

### 任务

配置 mirror 优先策略，并记录何时不应使用 `direct` 回退。

### 预期结果

从 mirror 成功初始化，并生成 provider installation 策略文件。

## 5. 运行手册

```sh
cd /workspace/practice/labs/12
terraform providers mirror /workspace/practice/labs/12/mirror
rm -rf .terraform .terraform.lock.hcl
TF_CLI_CONFIG_FILE=/workspace/practice/labs/12/terraform-cli.rc terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

关键结果：

- mirror 生成成功。
- `terraform init` 通过本地 mirror 安装 provider。
- 出现 mirror 相关 lock file checksum 警告，符合预期。
- `terraform test` 返回 `1 passed, 0 failed`。

## 7. 排障指南

1. mirror 缺 provider：重新执行 `terraform providers mirror`。
2. direct 误用：air-gapped 环境不要依赖 direct。
3. provider 未认证提示：这是本地 mirror 安装常见提示，需结合企业校验流程处理。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -rf .terraform mirror output tfplan terraform.tfstate*
```
