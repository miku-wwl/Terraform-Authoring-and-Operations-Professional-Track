# Terraform 实操训练 40：文件存在性 precondition

## 1. 课程元数据

- 课程编号：40
- 来源文件：`doc/40.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`fileexists()`、多条件 precondition、依赖文件
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Precondition 很适合检查部署前置条件，例如凭据文件是否存在、配置是否非空。注意 precondition 在资源创建前执行，因此不能依赖同一次 apply 才创建的文件。

## 3. 实验设计

背景：应用配置必须依赖已有数据库凭据文件。

问题：凭据文件缺失时不应生成应用配置。

任务：检查 `input/db.txt` 存在且非空。

预期结果：前置文件存在时创建 `output/app.txt`。

## 4. 实验文件

- `practice/labs/40/main.tf`
- `practice/labs/40/input/db.txt`
- `practice/labs/40/tests/file_precondition.tftest.hcl`
- `practice/labs/40/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/40
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。首次实现把 `db.txt` 设计成同一次 apply 创建，precondition 因创建前检查而失败；已修正为实验前置文件并复测通过。

## 7. 排障指南

1. `db.txt` 缺失：恢复 `input/db.txt`。
2. 文件为空：写入必要配置。
3. 不要让 precondition 依赖同一次创建的资源。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf output .terraform .terraform.lock.hcl
```
