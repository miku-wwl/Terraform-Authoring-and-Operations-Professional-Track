## 2. 知识点讲解

本节关注 HCP Terraform 的定价和功能层级。课程中强调：HCP Terraform 不是“所有能力都免费”的平台，不同组织会根据资源数量、使用方式和治理能力选择不同计划。实际价格和套餐名称可能随时间变化，复习时要以当前官方页面为准；考试更关注“哪些能力通常属于更高级治理能力”，而不是背某个固定金额。

核心理解有三层。第一，HCP Terraform 有托管服务和企业自托管方向，适用场景不同。第二，部分能力和资源数量、托管资源规模、组织计划有关。第三，像 audit logging、drift detection、continuous validation、policy governance、airgap installation 这类能力通常更偏企业治理，不应默认认为所有免费账号都能完整测试。

考试里容易出现的陷阱是把 HCP Terraform 的 UI、workspace 和 remote state 当作“完全免费且无限制”的功能，或者把 Terraform Enterprise 与 HCP Terraform 托管服务混为一谈。你需要能判断：个人学习可以理解概念，但企业落地时必须考虑计费、权限、审计和合规要求。

## 3. 考试选择题

1. 关于 HCP Terraform 定价，哪项理解最稳妥？
   - A. 所有功能永久免费
   - B. 所有功能只能购买企业版才能使用
   - C. 不同计划包含不同能力，企业治理能力通常依赖更高层级
   - D. 定价只和本地 Terraform CLI 版本有关

2. 哪类能力通常更偏企业治理和合规场景？
   - A. `terraform fmt`
   - B. audit logging、drift detection、policy governance
   - C. 本地变量默认值
   - D. `terraform console`

3. Airgap installation 更接近哪种场景？
   - A. 无需联网或强隔离环境中的企业自托管部署
   - B. 普通本地 Docker 练习
   - C. `terraform validate` 的默认功能
   - D. 每个免费 workspace 的必选设置

## 4. 判断题

1. 复习 HCP Terraform 定价时，应把具体价格视为可能变化的信息。
2. 企业级审计、漂移检测和持续验证等功能通常是考试中需要识别的治理能力。
3. Terraform Enterprise 和托管的 HCP Terraform 在部署与运营责任上完全相同。

## 5. 场景题

1. 公司要求所有基础设施变更必须满足审计留痕、策略控制和漂移检测。你会把这些需求归类为普通 CLI 能力，还是 HCP Terraform 的平台治理能力？请说明原因。

2. 个人学习者没有付费计划，但课程讲到了 drift detection。面对这种情况，应该如何安排复习？

## 6. 参考答案与解析

选择题：1-C，2-B，3-A。

判断题：1-对，2-对，3-错。

场景题参考：

1. 这些需求属于平台治理能力。CLI 可以执行 Terraform，但审计、集中审批、策略治理、漂移检测和团队权限通常需要平台或自建系统支撑。
2. 不必为了选择题强行购买付费能力。应理解 drift detection 的目标、触发逻辑、和 `plan` 检测差异的关系，再通过题目训练概念判断。

## 7. 复习提醒

- 不背固定价格，背能力分类。
- 区分“个人可体验功能”和“企业治理功能”。
- 遇到计划层级题，优先从治理、合规、审计、隔离部署等关键词判断。
