# 第 76 节：新式 S3 Backend Lockfile

## 知识点讲解

本节扩展 `doc/76.txt` 的 S3 backend 锁主题。课程原始内容讲的是 S3 backend 结合 DynamoDB table 做 state locking；Terraform 1.14 已经提示 `dynamodb_table` 参数 deprecated，新项目推荐使用 S3 backend 自带的 `use_lockfile`。

新式方案是：

- S3 bucket 保存 `terraform.tfstate`
- S3 backend 通过 `use_lockfile = true` 创建锁文件
- 不再需要 DynamoDB table

旧式 DynamoDB 锁仍然需要会读、会维护；新式 lockfile 则更适合新项目。第 75 节练旧式，本节练新式。

## 实验目标

1. 能区分 `dynamodb_table` 与 `use_lockfile` 两种锁方案。
2. 能在 LocalStack 上初始化使用 `use_lockfile` 的 S3 backend。
3. 能验证 state 写入远端 S3。
4. 能完成 `plan / apply / state list / destroy / clean` 闭环。
