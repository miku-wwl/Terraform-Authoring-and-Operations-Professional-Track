- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

本节从实际配置角度讲 plugin cache。可以通过 CLI 配置文件中的 `plugin_cache_dir`，也可以通过环境变量 `TF_PLUGIN_CACHE_DIR` 启用缓存。

新版本 Terraform 更重视 dependency lock file。如果缓存中的 provider 与 lock file 校验和不匹配，Terraform 可能不会复用缓存。团队需要决定是否提交 lock file，以及是否使用 `plugin_cache_may_break_dependency_lock_file`。

## 3. 实验设计

### 背景

团队要把 plugin cache 配置标准化，便于 Windows、Linux、CI runner 共用同一套原则。

### 问题

只创建缓存目录不够，还需要明确 CLI 配置、环境变量和 lock file 的关系。

### 任务

生成 Terraform CLI 配置示例，并用测试验证实践步骤包含 lock file 说明。

### 预期结果

生成 `output/terraformrc-plugin-cache.example`。

## 5. 运行手册

```sh
cd /workspace/practice/labs/10
mkdir -p .terraform-plugin-cache
export TF_PLUGIN_CACHE_DIR=/workspace/practice/labs/10/.terraform-plugin-cache
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 7. 排障指南

1. 缓存不生效：确认环境变量或 CLI 配置文件路径。
2. 多平台失败：检查 lock file 是否包含目标平台校验和。
3. CI 中路径错误：使用容器内绝对路径。
