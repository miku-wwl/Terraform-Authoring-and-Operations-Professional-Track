# Terraform 实操训练 30：从 JSON 中提取指定字段

## 1. 课程元数据

- 课程编号：30
- 来源文件：`doc/30.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`jsondecode`、索引访问、对象数组、字段筛选
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

解码 JSON 之后，Terraform 可以像处理普通 object/list 一样访问字段：对象用 `.key`，列表用 `[index]`，复杂数组可以配合 `for` 表达式筛选。

生产中很少需要整个 JSON，更多是从外部目录中提取某些字段，再传给资源、模块或输出。

## 3. 实验设计

背景：外部系统提供一个服务目录 JSON。

问题：Terraform 只需要 backend 服务和它的端口。

任务：读取 `catalog.json`，筛选 `tier == "backend"` 的服务，并生成摘要。

预期结果：输出服务名 `payments`、第一个端口 `8080` 和端口数量。

## 5. 运行手册

```sh
cd /workspace/practice/labs/30
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

已执行 `init/fmt/validate/test/plan/apply/output/destroy`。`terraform test` 返回 `1 passed, 0 failed`，成功筛选 backend 服务 `payments`，读取第一个端口 `8080`，端口数量为 `2`。

## 7. 排障指南

1. 索引越界：确认筛选结果非空。
2. 字段名错误：JSON key 区分大小写。
3. 端口 join 失败：数字列表需要先转换为字符串。
4. 结构变化：为外部 JSON 增加测试或变量校验。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf output .terraform .terraform.lock.hcl
```
