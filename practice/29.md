- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

`jsonencode` 将 Terraform 的 list、map、object 等数据结构编码成 JSON 字符串，适合传给外部系统或写入策略文件。`jsondecode` 将 JSON 字符串解析成 Terraform 原生数据结构，便于筛选、索引和组合。

生产中常见场景包括策略文档、服务目录、IP 范围清单和外部平台返回的 JSON 数据。

## 3. 实验设计

背景：团队需要生成服务 JSON，同时读取外部服务描述。

问题：手写 JSON 容易出错，外部 JSON 又需要被 Terraform 使用。

任务：编码 Terraform object，并解码 `data/service.json`。

预期结果：生成 JSON 文件，输出解码后的服务名和技能数量。

## 7. 排障指南

1. JSON 解析失败：检查文件是否是合法 JSON。
2. 字段访问失败：确认解码后的对象层级。
3. 数字和字符串混淆：注意 JSON 类型会映射到 Terraform 类型。
4. 手写 JSON 转义复杂：优先用 `jsonencode`。
