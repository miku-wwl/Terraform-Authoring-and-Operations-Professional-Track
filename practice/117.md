## 2. 知识点讲解

HCP Terraform 是 HashiCorp 提供的 Terraform 托管平台，课程中也会看到旧名称 Terraform Cloud。考试中遇到这两个名称时，要理解它们指向的是同一类托管协作能力，只是命名经历过变化。

传统 CLI 工作流里，工程师在本地运行 `terraform plan`、`terraform apply`、`terraform destroy`，输出、审批、变量管理和 state 管理往往依赖团队自己约定。HCP Terraform 的价值在于把这些动作放到一个统一平台里，让团队可以看到 run 历史、plan 输出、apply 审批人、state 版本、变量配置、VCS 触发来源、策略检查和成本估算。

从考试角度看，重点不是背 UI 菜单，而是理解平台解决了什么问题。它解决的是多人协作、审计追踪、远程 state、统一运行环境、敏感变量管理、策略治理和工作流标准化。比如同一个 workspace 可以记录每次 run，安全团队可以用 policy 阻止不合规资源，平台团队可以让代码从 GitHub 触发计划和审批。

常见混淆点是把 HCP Terraform 理解成 Terraform CLI 的替代品。更准确地说，它是 Terraform 运行和协作平台。CLI 仍然重要，很多场景还会通过 CLI-driven workflow 使用本地命令触发远端运行。

## 3. 考试选择题

1. 关于 HCP Terraform 的主要作用，哪项描述最准确？
   - A. 替代 Terraform 语言本身，让用户不再编写 `.tf` 文件
   - B. 提供托管的运行、协作、state、审批、变量和治理能力
   - C. 只用于创建 AWS EC2，不能管理其他云平台
   - D. 只用于本地调试，不适合团队协作

2. 在 HCP Terraform 中，run 历史对生产团队的价值主要是什么？
   - A. 自动删除所有旧 state
   - B. 记录计划、应用、审批和输出，便于审计与排障
   - C. 强制所有用户共享同一个本地目录
   - D. 禁止使用 Git 仓库存储 Terraform 代码

3. HCP Terraform 中的 cost estimation 和 policy check 一般位于哪个流程附近？
   - A. 代码格式化之前
   - B. plan 之后、apply 之前
   - C. destroy 之后
   - D. Terraform init 之前

## 4. 判断题

1. HCP Terraform 可以帮助团队保存 run 历史，因此比单纯本地 CLI 更适合审计。
2. 使用 HCP Terraform 后，Terraform CLI 就完全没有学习价值了。
3. HCP Terraform 可以通过 VCS 集成，在代码变更后触发 Terraform 运行流程。

## 5. 场景题

1. 一个团队中多名工程师各自在笔记本运行 `terraform apply`，几天后无法确认某次基础设施变更是谁审批和执行的。请说明 HCP Terraform 能缓解哪些问题。

2. 安全团队要求所有 EC2 都必须打标签，否则不能进入 apply 阶段。请说明 HCP Terraform 中哪个方向的能力适合承载这类控制。

## 6. 参考答案与解析

选择题：1-B，2-B，3-B。

判断题：1-对，2-错，3-对。

场景题参考：

1. HCP Terraform 可以集中保存 run 历史、plan 输出、apply 结果、审批记录和 state 版本，降低多人本地执行造成的审计缺口。
2. 这类要求适合用 policy as code，例如 Sentinel policy，在 plan 后、apply 前做组织策略检查。
