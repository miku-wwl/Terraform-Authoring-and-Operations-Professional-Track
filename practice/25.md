# Terraform 实操训练 25：Quoted string 与转义序列

## 1. 课程元数据

- 课程编号：25
- 来源文件：`doc/25.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：quoted string、`\"`、`\\`、`\n`
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Terraform 的双引号字符串支持转义序列。需要在字符串中保留双引号时使用 `\"`，保留反斜杠时使用 `\\`，换行可以使用 `\n`。这些细节在生成配置文件、脚本、路径和策略文本时很常见。

转义错误通常会在 `terraform validate` 阶段暴露，也可能生成语法正确但内容错误的文件。

## 3. 实验设计

背景：团队要生成一份包含引号、Windows 路径和多行说明的文件。

问题：普通字符串容易因为引号或反斜杠解析失败。

任务：使用转义序列生成 `escaped-strings.txt`。

预期结果：输出保留双引号和反斜杠。

## 4. 实验文件

- `practice/labs/25/main.tf`
- `practice/labs/25/tests/escape_sequences.tftest.hcl`
- `practice/labs/25/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/25
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

已执行 `init/fmt/validate/test/plan/apply/output/destroy`。`terraform test` 返回 `1 passed, 0 failed`，输出确认双引号和 Windows 路径反斜杠被正确保留。

## 7. 排障指南

1. 字符串提前结束：检查内部双引号是否使用 `\"`。
2. 路径少了反斜杠：检查是否使用 `\\`。
3. 文件没有换行：确认使用 `\n` 或 heredoc。
4. 内容复杂难读：考虑改用 heredoc 或 `templatefile`。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf output .terraform .terraform.lock.hcl
```
