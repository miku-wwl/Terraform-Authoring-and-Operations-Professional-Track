# Terraform 实操训练 37：密码长度验证

## 1. 课程元数据

- 课程编号：37
- 来源文件：`doc/37.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`validation`、`length()`、敏感变量
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

变量验证由 `validation`、`condition`、`error_message` 三部分组成。密码这类输入适合用长度、复杂度和非空规则做基础防线，同时要标记为 sensitive，避免在 CLI 中直接展示。

## 3. 实验设计

背景：数据库密码必须满足最低长度要求。

问题：短密码会带来安全风险。

任务：对 `db_password` 使用 `length()` 校验。

预期结果：默认密码通过，短密码在 plan 阶段失败。

## 4. 实验文件

- `practice/labs/37/main.tf`
- `practice/labs/37/tests/password_validation.tftest.hcl`
- `practice/labs/37/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/37
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。完整流程通过，测试确认最小长度为 12。

## 7. 排障指南

1. 输出显示 sensitive：这是预期保护。
2. 短密码失败：改为至少 12 位。
3. 自动化中不要交互输入：使用变量文件或环境变量传值。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
