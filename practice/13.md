# Terraform 实操训练 13：静态分析工具基础

## 1. 课程元数据

- 课程编号：13
- 来源文件：`doc/13.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：静态分析、Checkov、Terraform IaC 安全扫描
- 前置条件：Docker、`bridgecrew/checkov:latest`

## 2. 知识点讲解

静态分析是在部署前扫描代码质量、安全和最佳实践问题。对于 Terraform，Checkov 可以在不连接真实平台 API 的情况下扫描 `.tf` 文件，提前发现不安全的资源配置。

这种检查应该尽早放到开发和 CI 阶段，而不是等到生产部署前才发现。

## 3. 实验设计

### 背景

平台团队要验证 Terraform 代码是否含有明显安全风险。

### 问题

示例代码故意包含不合规的 Kubernetes Pod 配置，例如宿主机网络、特权容器和 `latest` 镜像标签。

### 任务

用 Checkov 扫描目录，观察 failed checks。

### 预期结果

Checkov 能识别 Kubernetes Pod 的安全上下文和运行配置问题。

## 4. 实验文件

- `practice/labs/13/main.tf`
- `practice/labs/13/README.md`

## 5. 运行手册

```powershell
docker run --rm `
  -v "${PWD}/practice/labs/13:/tf" `
  bridgecrew/checkov:latest `
  -d /tf --framework terraform --soft-fail --compact
```

## 6. 验证报告

状态：通过。

关键结果：

- Checkov 版本：`3.3.5`。
- 扫描返回 `Passed checks: 14, Failed checks: 13`。
- 关键失败项包括宿主机网络、特权容器、资源 requests/limits、`latest` 镜像标签等。
- 使用 `--soft-fail`，扫描发现问题但命令返回成功，适合作为教学实验。

## 7. 排障指南

1. 镜像不存在：先拉取 `bridgecrew/checkov:latest`。
2. 扫描失败但看不到结果：检查 volume 是否挂载到 `/tf`。
3. CI 要阻断部署：移除 `--soft-fail`。

## 8. 清理方式

本实验不创建 Terraform 资源，无需 destroy。
