# Terraform 考试训练 142：AWS CLI `source_profile` 配置讲解版

## 1. 课程定位

本节主题是 **AWS CLI `source_profile` 配置**。

这节不是让你去真实 AWS 账号里配置复杂 IAM，而是用 LocalStack + Terraform AWS Provider 的方式，理解 AWS Provider 在读取 profile、调用 API、维护 state、执行 plan/apply/destroy 时的基本路径。

原始课程给出的实验目标包括：

- 创建 AWS CLI base profile。
- 创建通过 `source_profile` 继承凭证的 lab profile。
- 理解 `source_profile` 与 assume role 配置的关系。

同时课程也明确提醒：LocalStack 使用 `http://localhost:4566` 作为 AWS API endpoint，凭证固定为 `test/test`，它适合训练 Terraform 写法、依赖关系、状态变化和输出设计，但不能替代真实 AWS 的 IAM 权限评估、真实扩缩容或真实网络连通性。

---

## 2. 一句话讲清楚 `source_profile`

`source_profile` 的作用是：

> 告诉 AWS CLI：当某个 profile 需要 assume role 时，先用哪个“基础 profile”的凭证去调用 STS AssumeRole。

也就是说，`source_profile` 不是单纯的“复制配置”，也不是普通意义上的“继承所有字段”。

它更像是：

> lab profile 自己没有长期 access key，它要通过 base profile 的凭证去换一组临时凭证。

---

## 3. base profile 和 lab profile 的关系

可以把它理解成两层身份。

### 3.1 base profile：基础身份

base profile 通常保存一组可用的基础凭证，例如：

```ini
[base]
aws_access_key_id = test
aws_secret_access_key = test
```

在真实 AWS 中，这里可能是 IAM User 的长期凭证，也可能来自 SSO、credential process、环境变量等。

在 LocalStack 练习里，凭证通常固定为：

```text
access key = test
secret key = test
```

因为 LocalStack 关注的是 API 兼容和 Terraform 行为练习，不是真的验证 AWS 账号身份。

### 3.2 lab profile：目标角色身份

lab profile 通常放在 AWS config 文件中，包含 `role_arn` 和 `source_profile`：

```ini
[profile lab]
role_arn = arn:aws:iam::000000000000:role/lab-role
source_profile = base
region = us-east-1
```

这段配置的意思是：

1. 使用 `base` profile 的凭证。
2. 调用 STS AssumeRole。
3. 尝试扮演 `lab-role`。
4. 后续用临时凭证访问 AWS API。

考试判断时，看到 `role_arn + source_profile`，你应该立刻联想到：

> 这是 assume role profile，不是普通静态凭证 profile。

---

## 4. `source_profile` 不等于“继承所有配置”

这是考试和实操里很容易误解的地方。

`source_profile = base` 主要解决的是：

> lab profile 用谁的凭证去发起 AssumeRole。

它不是说 lab profile 会自动继承 base profile 的所有配置，比如 region、output、endpoint 等。

所以在实际配置中，目标 profile 通常仍然需要自己声明必要字段，例如：

```ini
[profile lab]
role_arn = arn:aws:iam::000000000000:role/lab-role
source_profile = base
region = us-east-1
```

考试里如果看到这种说法：

> 只要配置了 `source_profile`，目标 profile 就会继承 base profile 的所有设置。

这个说法不稳。`source_profile` 的核心语义是凭证来源，不是完整配置继承。

---

## 5. `source_profile` 与 assume role 的关系

`source_profile` 通常和 `role_arn` 一起出现。

典型结构是：

```ini
[profile lab]
role_arn = arn:aws:iam::123456789012:role/AdminRole
source_profile = base
```

这里的关键逻辑是：

```text
base profile 的凭证
        ↓
调用 STS AssumeRole
        ↓
获得目标 role 的临时凭证
        ↓
使用临时凭证访问 AWS API
```

所以你可以把它记成：

> `source_profile` 管“从哪里来”，`role_arn` 管“要变成谁”。

---

## 6. Terraform AWS Provider 如何使用 profile

Terraform AWS Provider 可以通过 `profile` 指定使用哪个 AWS CLI profile。

示例：

```hcl
provider "aws" {
  region  = "us-east-1"
  profile = "lab"

  endpoints {
    iam = "http://localhost:4566"
    sts = "http://localhost:4566"
  }

  skip_credentials_validation = true
  skip_metadata_api_check     = true
  skip_requesting_account_id  = true
}
```

当 `profile = "lab"` 时，Provider 会尝试读取本机 AWS profile 配置。

如果 `lab` profile 是 assume role profile，那么 Provider 的身份链路大致是：

```text
Terraform AWS Provider
        ↓
读取 profile = lab
        ↓
发现 lab 有 role_arn + source_profile
        ↓
读取 source_profile = base
        ↓
使用 base 凭证进行 AssumeRole
        ↓
使用 assume role 后的临时凭证访问 AWS API
```

在真实 AWS 里，这一步会受到 IAM policy、trust policy、STS、账号权限等影响。

在 LocalStack 里，重点是验证 Terraform 是否能正确读取配置、发起调用、完成 plan/apply/destroy 的闭环。

## 7. LocalStack 练习边界

本节课程分类是：

```text
aws-localstack-limited
```

这说明它不是完全等价于真实 AWS 的实验。

LocalStack 可以帮助你练：

- Terraform AWS Provider 基本配置。
- profile 字段如何写。
- endpoint 如何指向本地 AWS API 模拟服务。
- 资源声明、查询、输出、销毁流程。
- Terraform state 如何记录资源。

但它不能完整替代真实 AWS 的：

- IAM trust policy 真实评估。
- STS AssumeRole 的完整安全语义。
- 跨账号角色扮演。
- 企业级权限边界。
- 真实 CloudTrail 审计。
- 真实 AWS 账号的安全责任模型。

所以这节课的重点不是证明你真的能跨账号 assume role，而是理解配置关系和 Terraform Provider 的读取路径。

## 8. 最小可理解配置结构

一个典型练习可以拆成三个部分。

### 8.1 credentials 文件

路径通常是：

```text
~/.aws/credentials
```

内容示例：

```ini
[base]
aws_access_key_id = test
aws_secret_access_key = test
```

### 8.2 config 文件

路径通常是：

```text
~/.aws/config
```

内容示例：

```ini
[profile lab]
role_arn = arn:aws:iam::000000000000:role/lab-role
source_profile = base
region = us-east-1
```

注意：在 `config` 文件里，非 default profile 通常写成：

```ini
[profile profile-name]
```

而在 `credentials` 文件里，通常直接写：

```ini
[profile-name]
```

这也是实操里非常容易写错的点。

### 8.3 Terraform provider

```hcl
provider "aws" {
  region  = "us-east-1"
  profile = "lab"

  endpoints {
    iam = "http://localhost:4566"
    sts = "http://localhost:4566"
  }

  skip_credentials_validation = true
  skip_metadata_api_check     = true
  skip_requesting_account_id  = true
}
```

在 LocalStack 练习中，经常需要配合一些 skip 参数，避免 Provider 去真实 AWS 做账号校验或元数据查询。

## 9. 和环境变量的区别

AWS 身份来源有很多种，常见包括：

- 环境变量：`AWS_ACCESS_KEY_ID`、`AWS_SECRET_ACCESS_KEY`。
- AWS credentials 文件。
- AWS config 文件。
- `AWS_PROFILE`。
- Provider 里的 `profile`。
- Assume role 配置。

考试里要知道，`source_profile` 属于 AWS CLI/shared config 的 profile 机制，不是 Terraform 变量，也不是 `.tfvars` 里的普通输入变量。

如果 Provider 里写了：

```hcl
profile = "lab"
```

那 Terraform 的 AWS Provider 会按 AWS SDK/profile 规则去找 `lab` 相关配置。

如果终端里设置了：

```bash
export AWS_PROFILE=lab
```

那很多 AWS CLI 或 SDK 调用也会默认使用 lab profile。

但考试不会要求你死背所有优先级细节，更重要的是理解：

> profile 是身份选择机制，`source_profile` 是 assume role 时的凭证来源。

## 10. 常见错误排查

### 错误 1：把 `[profile lab]` 写进 credentials 文件

错误示例：

```ini
[profile lab]
aws_access_key_id = test
aws_secret_access_key = test
```

在 credentials 文件里，通常不需要 `profile` 前缀。

更常见的写法是：

```ini
[lab]
aws_access_key_id = test
aws_secret_access_key = test
```

### 错误 2：lab profile 里只有 `source_profile`，没有 `role_arn`

错误示例：

```ini
[profile lab]
source_profile = base
```

这样不完整。`source_profile` 通常是为了配合 `role_arn` 使用。

更合理的写法：

```ini
[profile lab]
role_arn = arn:aws:iam::000000000000:role/lab-role
source_profile = base
```

### 错误 3：把 `source_profile` 理解成 Terraform provider 参数

错误示例：

```hcl
provider "aws" {
  source_profile = "base"
}
```

这不是 Terraform AWS Provider 的常规写法。

Terraform Provider 通常指定：

```hcl
provider "aws" {
  profile = "lab"
}
```

然后让 shared config 里的 `lab` profile 去处理 `source_profile` 和 `role_arn`。

### 错误 4：以为 LocalStack 能完整验证 IAM 权限边界

LocalStack 适合练 Terraform 流程，但它不能完整复刻真实 AWS 的 IAM 权限评估。

所以如果实验在 LocalStack 里能跑通，不代表真实 AWS 里一定能 assume role 成功。

真实 AWS 还需要满足：

- base identity 有权限调用 `sts:AssumeRole`。
- 目标 role 的 trust policy 信任 base identity。
- 目标 role 自己有对应权限。
- 组织 SCP、permission boundary 等没有阻止。

## 11. 考试判断题思路

### 题型 1：问 `source_profile` 的作用

稳妥答案：

> 指定 assume role 时使用哪个 profile 的凭证作为来源。

不要选：

- 创建 AWS 账号。
- 删除 Terraform state。
- 自动生成 IAM policy。
- 继承所有 profile 配置。

### 题型 2：问 `role_arn` 和 `source_profile` 的关系

稳妥答案：

> `role_arn` 指定要扮演的目标角色，`source_profile` 指定用于发起 AssumeRole 的基础凭证来源。

### 题型 3：问 LocalStack 能不能替代真实 AWS IAM

稳妥答案：

> 不能。LocalStack 可以用于本地 Terraform 路径练习，但不能完整替代真实 AWS IAM 权限、信任策略和跨账号行为。

## 13. 面试级理解

如果面试官问：

> `source_profile` 是干什么的？

可以这样答：

```text
source_profile is used in an AWS shared config profile that assumes a role. It tells the AWS CLI or SDK which base profile should provide the initial credentials for the STS AssumeRole call. The target profile usually contains role_arn and source_profile. In Terraform, the AWS provider can use the target profile, and the provider then follows the shared config chain to obtain temporary credentials. In LocalStack, we can practice the configuration flow, but real IAM permission and trust policy evaluation must be validated in an actual AWS account.
```

中文理解：

> `source_profile` 是 assume role 链路里的“起点凭证”。Terraform Provider 可以通过 `profile = "lab"` 使用这个链路，但真实权限是否能通过，要看 AWS IAM 和 STS 的真实评估。

## 14. 本节复习重点

考前只需要抓住五件事：

1. `source_profile` 通常和 `role_arn` 配合使用。
2. `source_profile` 指向基础凭证 profile。
3. `role_arn` 指向要扮演的目标 IAM role。
4. Terraform AWS Provider 可以通过 `profile` 使用 AWS CLI/shared config。
5. LocalStack 只能练配置和 Terraform 闭环，不能完整替代真实 IAM 权限验证。

## 16. 最终结论

本节的核心不是背 AWS CLI 配置文件格式，而是理解身份链路：

```text
Terraform provider profile = lab
        ↓
读取 AWS shared config 中的 lab profile
        ↓
lab profile 发现 role_arn + source_profile
        ↓
用 base profile 的凭证调用 AssumeRole
        ↓
拿到目标 role 的临时凭证
        ↓
访问 AWS API
```

考试中只要看到 `source_profile`，优先想到：

> 它是 assume role 时的基础凭证来源。

不要把它误解成 Terraform 变量、资源依赖、普通配置继承，或者 LocalStack 中能完整验证真实 AWS IAM 安全行为。
