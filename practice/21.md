# Terraform 实操训练 21：Resource targeting 基础

## 1. 课程元数据

- 课程编号：21
- 来源文件：`doc/21.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`-target`、资源地址、局部 plan/apply
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Terraform 默认会合并同一目录下所有 `.tf` 文件并计算完整变更。`-target` 可以只针对指定资源地址执行 plan、apply 或 destroy。

生产中 targeting 应作为异常处理手段，例如修复某个关键资源或分阶段恢复故障，而不应成为常规部署流程。长期依赖 targeting 会隐藏依赖关系问题，让 state 和配置演进变得难以推理。

## 3. 实验设计

背景：同一目录中有多个本地文件资源。

问题：运维只想先创建发布说明，不想触碰其他资源。

任务：使用 `-target=local_file.release_note` 执行局部变更。

预期结果：target 命令被记录，完整 destroy 可清理所有资源。

## 4. 实验文件

- `practice/labs/21/main.tf`
- `practice/labs/21/tests/resource_targeting.tftest.hcl`

## 5. 运行手册

```sh
cd /workspace/practice/labs/21
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -target=local_file.release_note -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

已执行 `init/fmt/validate/test/plan/apply/output/destroy`，`terraform test` 返回 `1 passed, 0 failed`。额外验证 targeted workflow：`terraform plan -target=local_file.release_note` 只创建 `release-note.txt`，不会创建 `runbook.txt`，随后 `destroy` 清理成功。

## 7. 排障指南

1. target 地址错误：用 `terraform state list` 或配置中的资源地址核对。
2. 出现 targeting 警告：这是预期提示，说明本次不是完整计划。
3. 下游资源未创建：target 不会自动创建依赖目标资源的下游。
4. 长期需要 target：应拆分模块或调整依赖边界。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf output .terraform .terraform.lock.hcl
```
