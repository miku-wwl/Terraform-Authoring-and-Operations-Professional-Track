## 2. 知识点讲解

HCP Terraform 的基础层级可以理解为：user 属于 organization，organization 下有 project，project 用来组织多个 workspace，workspace 承载具体 Terraform 配置的运行、变量、state 和协作记录。考试中最重要的是 workspace，因为很多功能都围绕它展开。

Organization 是团队协作空间，也是课程中强调的计费边界。一个用户可以加入多个 organization，每个 organization 可以有不同的计划、团队和设置。Project 更像分组和管理边界，可以把多个相关 workspace 放在一起。Workspace 则对应一套 Terraform 工作负载，通常关联某个 VCS 仓库、工作目录、变量、state、运行历史和执行设置。

不要把 HCP Terraform workspace 和 Terraform CLI 的 local workspace 混为一谈。HCP Terraform workspace 是平台中的协作和运行单元；CLI workspace 是同一配置下的多个 state 命名空间。两者都叫 workspace，但考试会通过上下文区分。

## 3. 考试选择题

   - A. organization
   - B. `.terraform` 目录

   - D. `terraform.tfvars`

2. HCP Terraform workspace 通常包含哪些内容？
   - A. run 历史、变量、state、执行配置和代码来源

   - C. 只包含本地 shell 历史
   - D. 只包含 Docker 镜像层

3. 关于 project 的作用，哪项更准确？
   - A. 用于组织和分组相关 workspace
   - B. 替代 Terraform provider
   - C. 自动创建云资源
   - D. 只能存放一个用户

## 4. 判断题

1. 一个 organization 可以包含多个 project 和多个 workspace。
2. HCP Terraform workspace 与 Terraform CLI workspace 在语义上完全相同。
3. Workspace 是理解 HCP Terraform run、state 和变量的关键对象。

## 5. 场景题

1. 公司有网络、应用、安全三类 Terraform 配置。你会如何用 project 和 workspace 做基本组织？

2. 考题说“billing is managed at organization level”。你应该如何理解这句话？

## 6. 参考答案与解析

选择题：1-A，2-A，3-A。

判断题：1-对，2-错，3-对。

场景题参考：

1. 可以为不同业务域建立 project，例如平台基础设施、应用服务、安全合规；每个 project 下放多个 workspace，例如 network-dev、network-prod、app-prod。
2. 计费、计划和组织级管理通常绑定 organization，而不是单个本地目录或单个 `.tf` 文件。

## 7. 复习提醒

- 记住 organization、project、workspace 三层关系。
- 特别区分 HCP Terraform workspace 和 CLI workspace。
- 看到变量、state、run、VCS，大概率是在说 workspace。
