# Terraform 实操训练 46：Lifecycle meta-argument 总览

## 1. 课程元数据

- 课程编号：46
- 来源文件：`doc/46.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`create_before_destroy`、`prevent_destroy`、`ignore_changes`、`replace_triggered_by`
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Lifecycle meta-argument 可以改变资源默认生命周期行为。它适合处理替换顺序、防误删、忽略外部漂移和强制联动替换，但应谨慎使用并写清原因。

## 3. 实验设计

背景：平台团队要给 lifecycle 建立速查表。

问题：不同参数用途容易混淆。

任务：输出四个 lifecycle 参数和含义。

预期结果：测试确认四个参数都存在。

## 4. 实验文件

- `practice/labs/46/main.tf`
- `practice/labs/46/tests/lifecycle_overview.tftest.hcl`
- `practice/labs/46/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/46
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。完整流程通过，测试确认四个 lifecycle 参数均已记录。

## 7. 排障指南

1. 参数混用：先明确要改变哪种生命周期行为。
2. 防误删不生效：`prevent_destroy` 必须仍在配置中。
3. 忽略过多属性会掩盖漂移。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
