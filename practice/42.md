# Terraform 实操训练 42：Sensitive 参数

## 1. 课程元数据

- 课程编号：42
- 来源文件：`doc/42.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：sensitive、敏感输出、state 风险
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

`sensitive = true` 会减少 CLI 中的明文泄露，但它不是加密机制。敏感值仍可能进入 state，因此 state 后端、访问权限和日志策略都要按密钥材料处理。

## 3. 实验设计

背景：自动化中需要传入 API token。

问题：token 不应在 plan/apply 输出中明文展示。

任务：将输入变量和输出标记为 sensitive，并输出非敏感派生状态。

预期结果：token 被隐藏，`token_is_configured` 为 true。

## 4. 实验文件

- `practice/labs/42/main.tf`
- `practice/labs/42/tests/sensitive_values.tftest.hcl`
- `practice/labs/42/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/42
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform output api_token
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。完整流程通过，测试确认敏感 token 已配置。

## 7. 排障指南

1. 输出显示 `(sensitive value)`：这是预期。
2. 需要查看敏感输出：显式执行 `terraform output api_token`。
3. 不要把 state 提交到 Git。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
