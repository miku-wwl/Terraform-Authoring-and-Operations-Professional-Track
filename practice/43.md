# Terraform 实操训练 43：HashiCorp Vault 基础

## 1. 课程元数据

- 课程编号：43
- 来源文件：`doc/43.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：Vault、secret 管理、dev server、state 风险
- 前置条件：Docker、`hashicorp/terraform:1.11`、`hashicorp/vault:1.18`

## 2. 知识点讲解

Vault 用于集中管理密钥、token、证书等敏感数据。Terraform 可以和 Vault 集成，但读取到的 secret 仍可能进入 state，因此必须把 state 也当成敏感资产。

## 3. 实验设计

背景：团队准备把数据库凭据从代码仓库迁移到 Vault。

问题：密钥直接进入 Terraform 代码和日志会造成泄露。

任务：记录 Vault 治理原则，并用 Vault dev server smoke test 写入读取 secret。

预期结果：Terraform 测试通过，Vault CLI 可以写入并读取 `secret/db_creds`。

## 4. 实验文件

- `practice/labs/43/main.tf`
- `practice/labs/43/scripts/vault-dev-smoke.sh`
- `practice/labs/43/tests/vault_readiness.tftest.hcl`
- `practice/labs/43/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/43
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

```sh
docker run --rm --cap-add IPC_LOCK -v "${PWD}:/workspace" -w /workspace hashicorp/vault:1.18 sh practice/labs/43/scripts/vault-dev-smoke.sh
```

## 6. 验证报告

状态：通过。Terraform 完整流程通过，Vault CLI smoke test 成功写入并读取 secret。

## 7. 排障指南

1. Vault 镜像不存在：先拉取 `hashicorp/vault:1.18`。
2. dev server 未启动完成：等待几秒后再读写。
3. 不要在生产使用 dev token。

## 8. 清理方式

Vault smoke test 使用 `--rm` 容器并在脚本退出时停止 dev server；Terraform 清理使用：

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
