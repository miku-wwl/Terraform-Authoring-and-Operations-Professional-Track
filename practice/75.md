# 第 75 节：State Locking 基础

## 知识点讲解

本节来自 doc/75.txt。核心主题是：理解写操作加锁、锁失败即停止、force-unlock 的风险，并在远端后端环境中保留锁相关配置入口。

在真实 AWS 中，这类实验会使用 S3 bucket 存放 	erraform.tfstate，并在需要并发保护时使用 DynamoDB 表保存锁信息。本项目不直接操作真实 AWS，统一使用 Docker 启动 LocalStack 模拟 S3、DynamoDB 和 STS。

## 实验目标

1. 能解释本节课程中的 state/backend 机制。
2. 能在 LocalStack 上初始化 S3 backend。
3. 能执行 	erraform plan、	erraform apply、	erraform state 或对应验证命令。
4. 能完成销毁和清理，避免残留本地容器资源。
