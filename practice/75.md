# 第 75 节：旧式 S3 Backend 与 DynamoDB State Lock

## 知识点讲解

本节来自 `doc/75.txt`。核心主题是 Terraform state locking：当 Terraform 执行可能写入 state 的操作时，会先获取锁；如果获取锁失败，Terraform 不应该继续写入 state，否则可能造成 state 损坏。

课程里讲到的典型生产方案是：

- S3 bucket 保存 `terraform.tfstate`
- DynamoDB table 保存锁信息和 state digest
- 表的分区键必须是 `LockID`，类型为 `String`

这是旧式但非常常见的写法。Terraform 1.14 会提示 `dynamodb_table` 已 deprecated，但大量存量系统仍然使用它，所以本节保留旧系统训练。

## 实验目标

1. 能解释 state lock 的目的和失败行为。
2. 能创建 LocalStack S3 bucket 与 DynamoDB lock table。
3. 能用 `dynamodb_table` 初始化 S3 backend。
4. 能完成 `plan / apply / state list / destroy / clean` 闭环。
