# Terraform 实操训练 14：Checkov 容器化安装验证

## 1. 课程元数据

- 课程编号：14
- 来源文件：`doc/14.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：Checkov 安装、工具容器、CI 工具版本
- 前置条件：Docker、`bridgecrew/checkov:latest`、`hashicorp/terraform:1.11`

## 2. 知识点讲解

课程演示的是在 Ubuntu 服务器安装 Checkov。为了避免污染本机和提高可重复性，本实验把安装步骤折叠为容器化运行：工具版本由镜像提供，CI runner 只需要 Docker。

生产 CI 中建议固定 Checkov 镜像版本，不要长期使用 `latest`。

## 3. 实验设计

### 背景

平台团队要在流水线中加入 Checkov，但不希望每个 runner 手工安装 Python 包。

### 问题

手工安装步骤不稳定，且不同 runner 版本容易漂移。

### 任务

用 Checkov 容器验证工具可用性，并用 Terraform test 固化安装策略说明。

### 预期结果

`checkov --version` 可执行，Terraform 测试通过。

## 5. 运行手册

```powershell
docker run --rm bridgecrew/checkov:latest --version
```

```sh
cd /workspace/practice/labs/14
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
```

## 6. 验证报告

状态：通过。

关键结果：

- Checkov 容器返回版本 `3.3.5`。
- Terraform `init/fmt/validate/test` 通过。
- `terraform test` 返回 `1 passed, 0 failed`。

## 7. 排障指南

1. 公司网络无法拉镜像：需要内部镜像仓库。
2. 版本漂移：CI 中固定镜像 tag。
3. Terraform test 失败：检查安装说明列表是否包含固定版本提醒。

## 8. 清理方式

本实验不创建 Terraform 资源，无需 destroy。
