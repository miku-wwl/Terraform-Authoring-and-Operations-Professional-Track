# Terraform 实操训练 48：`prevent_destroy`

## 1. 课程元数据

- 课程编号：48
- 来源文件：`doc/48.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：防误删、`prevent_destroy`、state 清理
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

`prevent_destroy` 会阻止 Terraform 销毁资源。它适合保护数据库、关键配置等高风险对象。注意它只有在资源块仍存在于配置中时才生效，移除整个资源块并不会继续提供保护。

## 3. 实验设计

背景：关键配置文件不允许被自动销毁。

问题：误执行 destroy 应被阻止。

任务：创建受保护文件，验证 destroy 失败，再用 `terraform state rm` 做实验清理。

预期结果：destroy 被阻止，state 清理后本地文件删除。

## 4. 实验文件

- `practice/labs/48/main.tf`
- `practice/labs/48/tests/prevent_destroy.tftest.hcl`
- `practice/labs/48/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/48
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform destroy -auto-approve
terraform state rm local_file.critical_config
rm -f output/critical-config.txt
```

## 6. 验证报告

状态：通过。验证中 `terraform destroy` 按预期失败并提示 `prevent_destroy`，随后使用 `terraform state rm` 和删除本地文件完成清理。

## 7. 排障指南

1. destroy 失败：这是本实验预期。
2. 想清理实验：先 `terraform state rm`，再删本地文件。
3. 真实生产不要随意移除 state，必须走变更审批。

## 8. 清理方式

```sh
terraform state rm local_file.critical_config
rm -f output/critical-config.txt
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
