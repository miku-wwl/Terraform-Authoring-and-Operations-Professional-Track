# Terraform 实操训练 8：`-no-color` 与机器可读日志

## 1. 课程元数据

- 课程编号：8
- 来源文件：`doc/8.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：`-no-color`、ANSI 控制符、CI 日志归档
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

Terraform 默认输出带颜色，适合人在终端阅读。但如果输出被重定向到文件、CI artifact 或安全扫描工具，颜色控制符会污染文本，使机器解析和审计困难。

`-no-color` 的价值是让输出变成纯文本，尤其适合 `plan` 审批、日志归档和第三方工具解析。

## 3. 实验设计

### 背景

平台团队要把 plan 输出作为审批材料归档。

### 问题

有颜色输出写入文件后会出现难读的控制字符。

### 任务

生成无颜色输出策略，并验证 CI plan 命令包含 `-input=false -no-color -out=tfplan`。

### 预期结果

生成 `output/no-color-policy.txt`，测试通过。

## 4. 实验文件

- `practice/labs/8/main.tf`
- `practice/labs/8/outputs.tf`

## 5. 运行手册

```sh
cd /workspace/practice/labs/8
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

关键结果：

- `terraform test` 返回 `1 passed, 0 failed`。
- `terraform plan` 显示 `Plan: 1 to add, 0 to change, 0 to destroy`。
- 本地策略文件创建后被成功销毁。

## 7. 排障指南

1. 日志有乱码：检查是否遗漏 `-no-color`。
2. 审批 artifact 无法阅读：不要保存默认彩色输出。
3. 测试失败：检查 `plan_capture_commands.ci_plan`。

## 8. 清理方式

```sh
terraform destroy -auto-approve
```
