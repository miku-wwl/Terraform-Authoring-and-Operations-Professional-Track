# 第 137 节：IAM Role 与 AssumeRole 信任策略

## 知识点讲解

本节对应 `doc/137.txt` 的课程录音转写。主题是 `IAM Role 与 AssumeRole 信任策略`，重点不是把真实 AWS 账号作为练习环境，而是在本地 LocalStack 中理解 Terraform AWS Provider 如何声明、查询、绑定和销毁 AWS 资源。

LocalStack 使用 `http://localhost:4566` 作为 AWS API endpoint，凭证固定为 `test/test`。这可以训练 Terraform 写法、依赖关系、状态变化和输出设计，但不能替代真实 AWS 的 IAM 权限评估、真实扩缩容或真实网络连通性。

## 实验目标

- 用 aws_iam_policy_document 编写 role trust policy。
- 创建允许 EC2 assume role 的 IAM role。
- 输出 role ARN 并理解信任策略的作用。

## 实验边界

- 不访问真实 AWS 账号。
- 不写入真实 access key、secret key 或 root 账号信息。
- 使用 Terraform AWS Provider + LocalStack 进行练习。
- 本节可在 LocalStack 中完成核心 Terraform 路径闭环。
