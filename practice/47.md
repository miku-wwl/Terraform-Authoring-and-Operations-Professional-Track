# Terraform 实操训练 47：`create_before_destroy`

## 1. 课程元数据

- 课程编号：47
- 来源文件：`doc/47.txt`
- 环境分类：`docker`
- 是否可用 Docker 运行：`yes`
- 核心主题：替换顺序、`create_before_destroy`、`triggers_replace`
- 前置条件：Docker、`hashicorp/terraform:1.11`

## 2. 知识点讲解

当资源必须替换时，Terraform 默认可能先销毁旧对象再创建新对象。`create_before_destroy` 会尝试先创建替代对象，再销毁旧对象，适合降低服务中断风险。

## 3. 实验设计

背景：服务镜像版本变化会触发替换。

问题：先销毁旧版本可能造成中断。

任务：用 `triggers_replace` 模拟替换，并启用 `create_before_destroy`。

预期结果：默认流程通过，并可用 `image_version=v2` 观察替换。

- `practice/labs/47/main.tf`

## 5. 运行手册

```sh
cd /workspace/practice/labs/47
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform plan -input=false -no-color -var 'image_version=v2' -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。完整流程通过，测试确认替换策略为 `create_before_destroy`。

## 7. 排障指南

1. 无法先创建：真实平台可能有唯一名称限制。
2. 替换未发生：确认触发替换的属性变化。
3. 替换成本高：先评估容量和配额。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf .terraform .terraform.lock.hcl
```
