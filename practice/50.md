## 2. 知识点讲解

本节把课程中的 list 数据类型 转成可运行的本地 Terraform 实验。重点不是背语法，而是理解它在生产环境中的建模价值、状态影响、测试方式和排障入口。

## 3. 实验设计

### 背景

平台团队需要把 list 数据类型 变成可重复执行的训练单元。

### 问题

只看字幕容易停留在概念层，缺少可验证的操作闭环。

### 任务

进入 practice/labs/50，运行 Terraform 命令并观察输出和测试断言。

### 预期结果

init/fmt/validate/test/plan/apply/output/destroy 全流程通过。

## 4. 实验文件

- `practice/labs/50/main.tf`
- `practice/labs/50/tests/`
- `practice/labs/50/README.md`

## 5. 运行手册

```sh
cd /workspace/practice/labs/50
terraform init -input=false
terraform fmt -check
terraform validate
terraform test
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
terraform destroy -auto-approve
```

## 6. 验证报告

状态：通过。

已在 Docker `hashicorp/terraform:1.11` 中执行完整流程。`terraform test` 返回 `1 passed, 0 failed`。本轮修复过的共性问题包括：部分生成文件需要 `terraform fmt` 标准化、测试框架实验需避免引用未声明的 local、`moved` block 需要写成真实 HCL block。

## 7. 排障指南

1. `terraform init` 失败：检查 provider 约束、模块路径和网络。
2. `terraform fmt -check` 失败：运行 `terraform fmt -recursive` 后再验证。
3. `terraform test` 失败：优先检查输出名称和断言条件。
4. `plan/apply` 不一致：检查是否有 apply 后才确定的值被用于 plan 阶段断言。

## 8. 清理方式

```sh
terraform destroy -auto-approve
rm -f tfplan
rm -rf output logs .terraform .terraform.lock.hcl terraform.tfstate terraform.tfstate.backup
```
