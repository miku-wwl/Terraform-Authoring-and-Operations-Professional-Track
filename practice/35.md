# Terraform AWS Sandbox 35：使用 `aws_ami` 创建 EC2

## 1. 课程元数据

- 课程编号：35
- 来源文件：`doc/35.txt`
- 环境分类：`aws-localstack-limited`
- 是否需要真实 AWS：`no`
- 是否已在 LocalStack 验证：`no`
- 涉及 AWS 服务：EC2、AMI
- 核心 Terraform 能力：`aws_ami` data source、资源引用、动态 AMI、EC2 创建
- 前置条件：HashiCorp Terraform Sandbox、LocalStack、Terraform CLI、AWS CLI

## 2. 知识点讲解

本节把 AMI data source 真正接到 `aws_instance` 上：先用 `data "aws_ami"` 查询镜像，再用 `data.aws_ami.latest.id` 作为 EC2 的 `ami` 参数。这样可以避免把 AMI ID 写死在资源块里。

真实 AWS 中还必须注意 architecture 和 instance type 是否匹配。例如 `arm64` AMI 不能随便配 `t3.micro`。LocalStack 不会完整模拟这类真实硬件约束，因此本实验只验证 Terraform authoring、data source 引用和 LocalStack EC2 创建路径。

## 3. 实验设计

背景：用户需要创建一台 EC2，但不允许硬编码 AMI ID。

任务：通过 `aws_ami` 查询最新模拟 AMI，并用它创建 `tf-lab-35-instance`。

预期结果：Terraform 创建一台 LocalStack EC2，输出 AMI ID 和 instance ID。

## 5. 运行手册

```sh
cd practice/labs/35
export AWS_ACCESS_KEY_ID=test
export AWS_SECRET_ACCESS_KEY=test
export AWS_DEFAULT_REGION=us-east-1
export LOCALSTACK_ENDPOINT=http://localhost:4566

bash scripts/check-sandbox.sh
bash scripts/bootstrap.sh
terraform init -input=false
terraform fmt -check
terraform validate
terraform plan -input=false -no-color -out=tfplan
terraform apply -auto-approve tfplan
terraform output
bash scripts/verify.sh
terraform destroy -auto-approve
bash scripts/clean.sh
```

## 6. 验证报告

状态：通过，且仍保持 `aws-localstack-limited` 分类。

已在 Windows 本机通过 Docker 启动 `localstack/localstack:4.2.0`，endpoint 为 `http://localhost:4566`。由于本机没有可用 `bash`，本次使用 PowerShell 等价执行 `bootstrap/verify/clean` 逻辑，并完整执行 Terraform 流程。

已执行命令摘要：

- `terraform init -input=false`
- `terraform fmt -check`
- `terraform validate`
- `terraform plan -input=false -no-color -out=tfplan`
- `terraform apply -auto-approve tfplan`
- `terraform output`
- `terraform destroy -auto-approve`

关键输出摘要：

- `data.aws_ami.latest` 成功读取模拟 AMI。
- `aws_instance.app` 成功创建 LocalStack EC2。
- `instance_id` 输出 1 个实例 ID。
- `selected_ami_id` 与 AWS CLI 查询到的实例 `ImageId` 一致。

限制说明：

- LocalStack 可以模拟 EC2 API，但不验证真实 AMI 目录、真实启动过程、网络连通性和计费行为。

清理结果：

- Terraform 创建的 `tf-lab-35-instance` 已 destroy。
- bootstrap 注册的 `tf-lab-app-*` 模拟 AMI 已 deregister。
- `.terraform/`、`.terraform.lock.hcl`、`tfplan`、`terraform.tfstate*` 已清理。

## 7. 排障指南

1. `aws_ami` 找不到镜像：先运行 `bootstrap.sh`。
2. EC2 创建失败：确认 provider endpoint 指向 LocalStack。
3. 真实 AWS 中架构不匹配：检查 AMI architecture 和 instance type。
4. 输出为空：确认资源引用的是 `data.aws_ami.latest.id`。

## 8. 清理方式

```sh
terraform destroy -auto-approve
bash scripts/clean.sh
rm -rf .terraform .terraform.lock.hcl tfplan terraform.tfstate terraform.tfstate.backup
```
